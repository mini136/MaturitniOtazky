Garbage Collector (GC)

Algoritmy GC

  • Mark & Sweep — označí dosažitelné objekty, ostatní uvolní
  • Generační GC — rozděluje objekty do generací (Gen0, Gen1, Gen2). Mladé objekty se kontrolují častěji.
  • Reference Counting (Python) — počítá počet referencí na objekt, při 0 uvolní. Problém s cyklickými referencemi.

Memory Leaks (úniky paměti)

  • Zapomenuté event handlery / listenery
  • Statické kolekce rostoucí bez omezení
  • Cyklické reference (v jazycích s ref counting)
  • Neuzavřené streamy/connections

Ukázkový kód — C# (hodnota vs reference, GC, IDisposable)

Ukázka hodnotových vs referenčních typů v C#. Struct Bod je hodnotový typ (kopie), class Osoba je referenční typ (sdílí data).
// === HODNOTOVE vs REFERENCNI TYPY ===
struct Bod {
    public int X, Y;
    public Bod(int x, int y) { X = x; Y = y; }
    public override string ToString() => $"({X}, {Y})";
}

class Osoba {
    public string Jmeno;
    public int Vek;
}

// Hodnotovy typ - kopie
Bod b1 = new Bod(1, 2);
Bod b2 = b1;        // KOPIE hodnot
b2.X = 99;
// b1.X je stale 1 — nezmenilo se!

// Referencni typ - sdilena reference
Osoba o1 = new Osoba { Jmeno = "Alice", Vek = 25 };
Osoba o2 = o1;      // SDILENA reference
o2.Vek = 30;
// o1.Vek je ted 30 — zmena se projevila!
Generace GC v C# — objekty se posouvají z Gen0 do Gen2. Metoda GC.Collect() vynutí GC.
// Generace GC
object obj = new object();
Console.WriteLine($"Generace: {GC.GetGeneration(obj)}"); // 0

GC.Collect();
Console.WriteLine($"Po GC: {GC.GetGeneration(obj)}");    // 1

GC.Collect();
Console.WriteLine($"Po 2x GC: {GC.GetGeneration(obj)}"); // 2
WeakReference umožňuje odkazovat na objekt, který může GC uvolnit. IDisposable vzor pro deterministické uvolnění prostředků.
// WeakReference - slaba reference (GC muze uvolnit)
var silna = new object();
var slaba = new WeakReference(silna);
silna = null;
GC.Collect();
Console.WriteLine($"Slaba reference zije: {slaba.IsAlive}"); // False

// IDisposable vzor — using blok zajisti uvolneni
class SpravceProstedku : IDisposable {
    private bool disposed = false;
    public void Dispose() {
        if (!disposed) {
            Console.WriteLine("  Uvolnuji prostredky...");
            disposed = true;
        }
        GC.SuppressFinalize(this);
    }
    ~SpravceProstedku() { Dispose(); }
}

using (var sp = new SpravceProstedku()) {
    Console.WriteLine("  Pouzivam prostredky...");
} // Dispose() se zavola automaticky

Ukázkový kód — Python (reference counting, cyklické reference)

Reference counting v Pythonu — sys.getrefcount() ukazuje počet referencí. Při dosažení 0 se objekt uvolní.
import sys

a = [1, 2, 3]
print(f"Pocet referenci: {sys.getrefcount(a)}")  # 2 (a + argument getrefcount)

b = a      # dalsi reference
print(f"Po b = a: {sys.getrefcount(a)}")         # 3

del b      # smazeme referenci
print(f"Po del b: {sys.getrefcount(a)}")         # 2
Identita objektu (id()) a rozdíl mezi mutable/immutable typy. Mutable objekty sdílejí identitu při přiřazení.
# Identita objektu
x = [1, 2, 3]
y = x            # stejna reference
z = [1, 2, 3]   # novy objekt

print(f"x is y: {x is y}")  # True  (stejna reference)
print(f"x is z: {x is z}")  # False (jina reference, stejny obsah)
print(f"x == z: {x == z}")  # True  (stejny obsah)

# Mutable vs Immutable
a = 42           # immutable (int)
b = a
b = 100          # vytvori NOVY objekt
print(f"a = {a}")  # 42 — nezmeneno

c = [1, 2, 3]   # mutable (list)
d = c
d.append(4)      # meni STEJNY objekt
print(f"c = {c}")  # [1, 2, 3, 4] — zmeneno!
Cyklické reference mezi objekty. Python gc modul dokáže detekovat a sbírat cyklické reference.
import gc

class Uzel:
    def __init__(self, nazev):
        self.nazev = nazev
        self.soused = None

# Cyklicka reference
a = Uzel("A")
b = Uzel("B")
a.soused = b
b.soused = a

del a, b  # reference counting nestaci (cyklus)!
gc.collect()  # cyklicky GC je sebere
