Scrum

Role

  • Product Owner — definuje priority, backlog
  • Scrum Master — odstraňuje překážky, facilituje
  • Dev Team — implementace (3–9 lidí)

Artefakty

  • Product Backlog — prioritizovaný seznam požadavků
  • Sprint Backlog — úkoly pro aktuální sprint
  • Increment — fungující přírůstek produktu

Ceremonie

  • Sprint Planning — výběr úkolů do sprintu
  • Daily Standup — 15min denní sync
  • Sprint Review — demo výsledků
  • Sprint Retrospective — co zlepšit

Kanban

  • Vizuální tabule: To Do → In Progress → Done
  • WIP limit — omezení počtu rozpracovaných úkolů
  • Continuous flow — bez pevných sprintů

DevOps & CI/CD

  • CI (Continuous Integration) — automatický build + testy při každém commitu
  • CD (Continuous Delivery/Deployment) — automatické nasazení
  • Nástroje: Jenkins, GitHub Actions, GitLab CI

Kanban tabule s WIP limitem — omezuje počet úkolů současně rozpracovaných. Pokud je limit překročen, úkol nelze přesunout do "In Progress".
class KanbanTabule:
    """Kanban tabule s WIP limitem"""

    def __init__(self, wip_limit=3):
        self.sloupce = {
            "Backlog":     [],
            "In Progress": [],
            "Review":      [],
            "Done":        []
        }
        self.wip_limit = wip_limit

    def pridej_ukol(self, ukol):
        self.sloupce["Backlog"].append(ukol)

    def presun(self, ukol, odkud, kam):
        if ukol not in self.sloupce[odkud]:
            print(f"  Ukol '{ukol}' neni v '{odkud}'")
            return False

        # WIP limit pro In Progress a Review
        if kam in ("In Progress", "Review"):
            if len(self.sloupce[kam]) >= self.wip_limit:
                print(f"  WIP limit ({self.wip_limit}) dosazeny pro '{kam}'!")
                return False

        self.sloupce[odkud].remove(ukol)
        self.sloupce[kam].append(ukol)
        return True

    def zobraz(self):
        print("  Kanban tabule:")
        for sloupec, ukoly in self.sloupce.items():
            print(f"    {sloupec:15s}: {ukoly}")

# Pouziti
tabule = KanbanTabule(wip_limit=2)
for ukol in ["Login page", "API endpoint", "Databaze", "Testy"]:
    tabule.pridej_ukol(ukol)

tabule.presun("Login page", "Backlog", "In Progress")
tabule.presun("API endpoint", "Backlog", "In Progress")
tabule.presun("Databaze", "Backlog", "In Progress")  # WIP limit!
tabule.presun("Login page", "In Progress", "Done")
tabule.zobraz()
Scrum Sprint simulace — UserStory s body, sprint s kapacitou (velocity). Sprint automaticky doplňuje úkoly z backlogu podle dostupné kapacity.
from dataclasses import dataclass

@dataclass
class UserStory:
    nazev: str
    body: int       # story points
    stav: str = "backlog"

class Sprint:
    def __init__(self, cislo, velocity=20):
        self.cislo = cislo
        self.velocity = velocity  # kapacita v bodech
        self.stories = []

    def pridej_story(self, story):
        aktualni = sum(s.body for s in self.stories)
        if aktualni + story.body <= self.velocity:
            story.stav = "sprint"
            self.stories.append(story)
            return True
        print(f"    Nelze pridat '{story.nazev}' ({story.body}b) - kapacita!")
        return False

    def dokonci(self):
        for s in self.stories:
            s.stav = "done"
        hotove = sum(s.body for s in self.stories)
        print(f"  Sprint {self.cislo}: dokonceno {hotove}/{self.velocity} bodu")

# Simulace
backlog = [
    UserStory("Login", 5), UserStory("Registrace", 8),
    UserStory("Profil", 3), UserStory("Dashboard", 13),
    UserStory("Export", 5)
]

sprint = Sprint(1, velocity=20)
print("  Sprint Planning:")
for story in backlog:
    vysledek = sprint.pridej_story(story)
    status = "PRIDANO" if vysledek else "NEZARAZENO"
    print(f"    {story.nazev} ({story.body}b): {status}")

sprint.dokonci()
