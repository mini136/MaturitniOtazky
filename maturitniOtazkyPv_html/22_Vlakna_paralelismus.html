⚡ ČÁST 2 — Concurrency vs Parallelism

Concurrency vs Parallelism

  • Concurrency — střídání úloh (i na 1 jádru), I/O-bound
  • Parallelism — skutečně současný běh na více jádrech, CPU-bound

GIL (Global Interpreter Lock): V CPythonu jen jedno vlákno najednou provádí Python bytecode. Pro CPU-bound úlohy použij multiprocessing!

Ukázkový kód — Python

Race condition — bez zámku dvě vlákna špatně aktualizují sdílenou proměnnou. S Lock je výsledek správný (200000).
import threading

# Race condition demo
pocitadlo_spatne = 0
pocitadlo_spravne = 0
zamek = threading.Lock()

def inkrementuj_spatne():
    global pocitadlo_spatne
    for _ in range(100_000):
        pocitadlo_spatne += 1   # RACE CONDITION!

def inkrementuj_spravne():
    global pocitadlo_spravne
    for _ in range(100_000):
        with zamek:             # MUTEX
            pocitadlo_spravne += 1

# Bez zamku
t1 = threading.Thread(target=inkrementuj_spatne)
t2 = threading.Thread(target=inkrementuj_spatne)
t1.start(); t2.start()
t1.join(); t2.join()
print(f"  Bez zamku: {pocitadlo_spatne} (ocekavano 200000)")

# Se zamkem
t1 = threading.Thread(target=inkrementuj_spravne)
t2 = threading.Thread(target=inkrementuj_spravne)
t1.start(); t2.start()
t1.join(); t2.join()
print(f"  Se zamkem: {pocitadlo_spravne}")
Semaphore — omezuje počet současně běžících vláken (zde max 3). Simuluje omezenou kapacitu zdroje (např. databázový connection pool).
import time

semafor = threading.Semaphore(3)  # max 3 soucasne

def pristup_k_db(id):
    print(f"  Vlakno {id}: ceka na pristup...")
    with semafor:
        print(f"  Vlakno {id}: pristup k DB")
        time.sleep(0.1)  # simulace prace
        print(f"  Vlakno {id}: hotovo")

vlakna = [threading.Thread(target=pristup_k_db, args=(i,)) for i in range(6)]
for v in vlakna: v.start()
for v in vlakna: v.join()
Producer-Consumer — klasický vzor s frontou (queue.Queue). Producent vyrábí úlohy, konzument je zpracovává. Fronta je thread-safe.
import queue

def producer(q, pocet):
    for i in range(pocet):
        item = f"uloha_{i}"
        q.put(item)
        print(f"  Produced: {item}")
    q.put(None)  # signal k ukonceni

def consumer(q):
    while True:
        item = q.get()
        if item is None:
            break
        print(f"  Consumed: {item}")
        q.task_done()

q = queue.Queue(maxsize=5)
prod = threading.Thread(target=producer, args=(q, 5))
cons = threading.Thread(target=consumer, args=(q,))
prod.start(); cons.start()
prod.join(); cons.join()
ThreadPoolExecutor — vysokoúrovňové API pro správu poolu vláken. map() paralelně zpracuje všechny vstupy a vrátí výsledky v pořadí.
from concurrent.futures import ThreadPoolExecutor

def zpracuj(cislo):
    """Simulace I/O operace"""
    time.sleep(0.05)
    return cislo ** 2

# ThreadPool - jednoduche pouziti
with ThreadPoolExecutor(max_workers=4) as executor:
    vysledky = list(executor.map(zpracuj, range(10)))
    print(f"  Vysledky: {vysledky}")
Multiprocessing — obchází GIL, každý worker běží v samostatném procesu. Vhodné pro CPU-bound úlohy (výpočty).
from multiprocessing import Pool

def cpu_narocna(n):
    """CPU-bound uloha"""
    return sum(i*i for i in range(n))

# Multiprocessing - obchazi GIL
if __name__ == "__main__":
    with Pool(processes=4) as pool:
        vstupy = [100_000, 200_000, 300_000, 400_000]
        vysledky = pool.map(cpu_narocna, vstupy)
        print(f"  Multiprocessing: {vysledky}")
⏱️ ČÁST 4 — Asynchronní metody a Concurrent design patterns

Asynchronní metody (async/await)

Asynchronní programování umožňuje spouštět operace bez blokování hlavního vlákna. Ideální pro I/O operace (síť, soubory, API).

KonceptPythonC#
Asynchonní funkceasync def f(): ...async Task F() { ... }
Await výsledkuawait asyncio.sleep(1)await Task.Delay(1000)
Spuštěníasyncio.run(main())await Task.WhenAll(t1, t2)
Paralelní spuštěníasyncio.gather(f1(), f2())Task.WhenAll(T1, T2)

async/await vs vlákna: async je kooperativní multitasking (jedno vlákno, přepnutí na await), vlákna jsou preemptivní (OS rozhoduje o přepnutí). Async je vhodné pro I/O-bound úlohy, vlákna pro CPU-bound.

Python async/await — soubněžné stažení dat bez blokování (asyncio.gather). Event loop spravuje korutiny.
import asyncio
import time

async def stahuj(url, delay):
    print(f"  Start: {url}")
    await asyncio.sleep(delay)  # simulace I/O (neblokujici)
    print(f"  Hotovo: {url}")
    return f"data z {url}"

async def main():
    start = time.time()
    # Paralelni spusteni - vsechny bezi soucasne
    vysledky = await asyncio.gather(
        stahuj("api.example.com", 0.3),
        stahuj("db.example.com", 0.2),
        stahuj("cdn.example.com", 0.1),
    )
    print(f"  Cas: {time.time()-start:.2f}s (ne 0.6s!)")
    return vysledky

asyncio.run(main())
C# async/await — asynchronní HTTP volání. await uvolní vlákno během čekání na odpověď.
using System.Net.Http;

async Task<string> StahujAsync(string url)
{
    using var client = new HttpClient();
    return await client.GetStringAsync(url);  // neblokuje vlakno
}

async Task HlavniAsync()
{
    // Paralelni spusteni
    var ukoly = new[]
    {
        StahujAsync("https://api1.example.com"),
        StahujAsync("https://api2.example.com")
    };
    string[] vysledky = await Task.WhenAll(ukoly);
    Console.WriteLine($"Stazeno {vysledky.Length} odpovedi");
}

Concurrent design patterns

VzorPopisImplementace
Producer-ConsumerProducent přidává úlohy, konzument je zpracováváqueue.Queue (Python), BlockingCollection (C#)
Thread PoolPool předpřipravených vláken pro opakované úlohyThreadPoolExecutor, Task.Run
Active ObjectMetody se spouštějí asynchronně v vlastním vlákněFronta požadavků + vlákno služby
Monitor (Guarded Suspension)Vlákno čeká na splnění podmínkyCondition.wait(), Monitor.Wait()
Read-Write LockVíce čtení současně, jeden zápis exkluzivněthreading.RLock, ReaderWriterLockSlim

Základní pravidlo: Producer-Consumer a Thread Pool jsou nejběžnější — Thread Pool (řeší práci více vlákny) + Producer-Consumer (odděluje produkci od spotřeby) jsou často kombinovány.
