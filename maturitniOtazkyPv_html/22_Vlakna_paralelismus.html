Concurrency vs Parallelism

  • Concurrency — střídání úloh (i na 1 jádru), I/O-bound
  • Parallelism — skutečně současný běh na více jádrech, CPU-bound

Synchronizační primitiva

GIL (Global Interpreter Lock): V CPythonu jen jedno vlákno najednou provádí Python bytecode. Pro CPU-bound úlohy použij multiprocessing!

Ukázkový kód — Python

Race condition — bez zámku dvě vlákna špatně aktualizují sdílenou proměnnou. S Lock je výsledek správný (200000).
import threading

# Race condition demo
pocitadlo_spatne = 0
pocitadlo_spravne = 0
zamek = threading.Lock()

def inkrementuj_spatne():
    global pocitadlo_spatne
    for _ in range(100_000):
        pocitadlo_spatne += 1   # RACE CONDITION!

def inkrementuj_spravne():
    global pocitadlo_spravne
    for _ in range(100_000):
        with zamek:             # MUTEX
            pocitadlo_spravne += 1

# Bez zamku
t1 = threading.Thread(target=inkrementuj_spatne)
t2 = threading.Thread(target=inkrementuj_spatne)
t1.start(); t2.start()
t1.join(); t2.join()
print(f"  Bez zamku: {pocitadlo_spatne} (ocekavano 200000)")

# Se zamkem
t1 = threading.Thread(target=inkrementuj_spravne)
t2 = threading.Thread(target=inkrementuj_spravne)
t1.start(); t2.start()
t1.join(); t2.join()
print(f"  Se zamkem: {pocitadlo_spravne}")
Semaphore — omezuje počet současně běžících vláken (zde max 3). Simuluje omezenou kapacitu zdroje (např. databázový connection pool).
import time

semafor = threading.Semaphore(3)  # max 3 soucasne

def pristup_k_db(id):
    print(f"  Vlakno {id}: ceka na pristup...")
    with semafor:
        print(f"  Vlakno {id}: pristup k DB")
        time.sleep(0.1)  # simulace prace
        print(f"  Vlakno {id}: hotovo")

vlakna = [threading.Thread(target=pristup_k_db, args=(i,)) for i in range(6)]
for v in vlakna: v.start()
for v in vlakna: v.join()
Producer-Consumer — klasický vzor s frontou (queue.Queue). Producent vyrábí úlohy, konzument je zpracovává. Fronta je thread-safe.
import queue

def producer(q, pocet):
    for i in range(pocet):
        item = f"uloha_{i}"
        q.put(item)
        print(f"  Produced: {item}")
    q.put(None)  # signal k ukonceni

def consumer(q):
    while True:
        item = q.get()
        if item is None:
            break
        print(f"  Consumed: {item}")
        q.task_done()

q = queue.Queue(maxsize=5)
prod = threading.Thread(target=producer, args=(q, 5))
cons = threading.Thread(target=consumer, args=(q,))
prod.start(); cons.start()
prod.join(); cons.join()
ThreadPoolExecutor — vysokoúrovňové API pro správu poolu vláken. map() paralelně zpracuje všechny vstupy a vrátí výsledky v pořadí.
from concurrent.futures import ThreadPoolExecutor

def zpracuj(cislo):
    """Simulace I/O operace"""
    time.sleep(0.05)
    return cislo ** 2

# ThreadPool - jednoduche pouziti
with ThreadPoolExecutor(max_workers=4) as executor:
    vysledky = list(executor.map(zpracuj, range(10)))
    print(f"  Vysledky: {vysledky}")
Multiprocessing — obchází GIL, každý worker běží v samostatném procesu. Vhodné pro CPU-bound úlohy (výpočty).
from multiprocessing import Pool

def cpu_narocna(n):
    """CPU-bound uloha"""
    return sum(i*i for i in range(n))

# Multiprocessing - obchazi GIL
if __name__ == "__main__":
    with Pool(processes=4) as pool:
        vstupy = [100_000, 200_000, 300_000, 400_000]
        vysledky = pool.map(cpu_narocna, vstupy)
        print(f"  Multiprocessing: {vysledky}")
