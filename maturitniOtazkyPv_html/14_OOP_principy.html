1. Zapouzdření (Encapsulation)

Vnitřní stav objektu je skrytý před okolím. Přístup k datům je možný jen přes veřejné metody nebo properties. Díky tomu lze validovat vstupy a chránit konzistenci objektu.

Modifikátory přístupu:
public — dostupné odkudkoliv  |  protected — dostupné v třídě a potomcích  |  private — dostupné jen uvnitř třídy
V Pythonu: _chraneny (konvence), __soukromy (name mangling → _Trida__soukromy)
Bankovní účet — __zustatek je soukromý, nelze ho nastavit přímo. Musíš projít metodou vloz() nebo vyber(), která provede validaci.
class BankovniUcet:
    def __init__(self, majitel, zustatek=0):
        self._majitel = majitel      # protected (konvence)
        self.__zustatek = zustatek   # private (name mangling)

    @property
    def zustatek(self):              # getter — cte hodnotu
        return self.__zustatek

    def vloz(self, castka):
        if castka <= 0:
            raise ValueError("Castka musi byt kladna!")
        self.__zustatek += castka

    def vyber(self, castka):
        if castka > self.__zustatek:
            raise ValueError("Nedostatek prostredku!")
        self.__zustatek -= castka

ucet = BankovniUcet("Alice", 1000)
ucet.vloz(500)
ucet.vyber(200)
print(ucet.zustatek)   # 1300
# ucet.__zustatek = 9999  ← AttributeError! (name mangling)

2. Dědičnost (Inheritance)

Potomek (child class) přebírá atributy a metody rodiče (parent class) a může je rozšiřovat nebo přepisovat. Umožňuje znovupoužitelnost kódu a budování hierarchií tříd.

Klíčové pojmy:
super().__init__() — volání konstruktoru rodiče  |  override — přepsání metody rodiče v potomkovi  |  MRO (Method Resolution Order) — pořadí hledání metody při vícenásobné dědičnosti v Pythonu
Hierarchie zvířat — Zivocich je základní třída. Pes dědí od ní, přidá породu a přepíše zvuk(). Vždy zavolej super().__init__() pro inicializaci rodiče.
class Zivocich:
    def __init__(self, jmeno, vek):
        self.jmeno = jmeno
        self.vek = vek

    def info(self):
        return f"{self.jmeno} ({self.vek} let)"

    def zvuk(self):
        return "..."

class Pes(Zivocich):            # dedi od Zivocich
    def __init__(self, jmeno, vek, pleme):
        super().__init__(jmeno, vek)   # zavolej rodicovsky init
        self.pleme = pleme

    def zvuk(self):             # override metody rodiče
        return "Haf!"

    def info(self):             # rozsireni metody rodiče
        return super().info() + f", pleme: {self.pleme}"

class Kocka(Zivocich):
    def zvuk(self): return "Mnau!"

pes = Pes("Rex", 3, "Labrador")
print(pes.info())   # Rex (3 let), pleme: Labrador
print(pes.zvuk())   # Haf!
print(isinstance(pes, Zivocich))  # True — pes IS-A Zivocich
⚠️ IS-A vs HAS-A: Dědičnost modeluje vztah „je" (Pes je Zvíře). Pokud vztah zní „má", použij kompozici/agregaci místo dědičnosti.

3. Polymorfismus

Stejné volání metody se chová různě podle skutečného typu objektu. Umožňuje psát obecný kód, který funguje s různými typy.

Druhy polymorfismu:
Runtime (dynamický) — přepsání metody v potomkovi (override), rozhoduje se za běhu  |  Compile-time (statický) — přetížení metody (overloading), různé parametry, v Pythonu není (duck typing)
Geometrické tvary — funkce plocha_info() volá plocha() na libovolném tvaru. Nezajímá ji konkrétní typ — stačí, že objekt má metodu plocha().
from abc import ABC, abstractmethod
import math

class Tvar(ABC):
    @abstractmethod
    def plocha(self) -> float: pass

    @abstractmethod
    def obvod(self) -> float: pass

    def popis(self):
        return f"{type(self).__name__}: plocha={self.plocha():.2f}, obvod={self.obvod():.2f}"

class Kruh(Tvar):
    def __init__(self, r): self.r = r
    def plocha(self): return math.pi * self.r ** 2
    def obvod(self): return 2 * math.pi * self.r

class Obdelnik(Tvar):
    def __init__(self, a, b): self.a, self.b = a, b
    def plocha(self): return self.a * self.b
    def obvod(self): return 2 * (self.a + self.b)

class Trojuhelnik(Tvar):
    def __init__(self, a, v): self.a, self.v = a, v
    def plocha(self): return 0.5 * self.a * self.v
    def obvod(self): return 3 * self.a  # zjednoduseno

# Polymorfismus — stejná funkce, různé chování
tvary = [Kruh(5), Obdelnik(4, 6), Trojuhelnik(3, 4)]
for t in tvary:
    print(t.popis())
# Kruh: plocha=78.54, obvod=31.42
# Obdelnik: plocha=24.00, obvod=20.00
# Trojuhelnik: plocha=6.00, obvod=9.00

4. Abstrakce

Uživatel třídy vidí jen co objekt dělá, ne jak to dělá. Abstrakce se realizuje pomocí abstraktních tříd a rozhraní (interface).

Abstraktní třída vs Interface:
Abstraktní třídaInterface
Může mít implementaci?✅ Ano (konkrétní metody)❌ Ne (jen signatury) — v Pythonu ABC s @abstractmethod
Vícenásobná dědičnost?Omezená (C#/Java: 1 třída)✅ Ano (libovolně mnoho)
Kdy použít?Sdílená implementace + přirozená hierarchieSmlouva chování bez hierarchie
Databázové připojení — Database je abstraktní třída definující kontrakt. Konkrétní implementace (MySQL, SQLite) skryjí detaily před kódem, který je volá.
from abc import ABC, abstractmethod

class Database(ABC):
    """Abstraktni trida — definuje kontrakt"""
    @abstractmethod
    def connect(self, url: str): pass

    @abstractmethod
    def query(self, sql: str) -> list: pass

    def execute_safe(self, sql: str):   # konkretni sdilena metoda
        if "DROP" in sql.upper():
            raise PermissionError("DROP neni povolen!")
        return self.query(sql)

class MySQLDatabase(Database):
    def connect(self, url): print(f"MySQL: pripojuji na {url}")
    def query(self, sql): return [{"id": 1}]  # simulace

class SQLiteDatabase(Database):
    def connect(self, url): print(f"SQLite: oteviram {url}")
    def query(self, sql): return [{"id": 2}]

def inicializuj(db: Database, url: str):
    db.connect(url)                          # nezajima nas typ
    return db.execute_safe("SELECT * FROM users")

db = MySQLDatabase()
vysledky = inicializuj(db, "localhost:3306")
# Stejnou funkci lze zavolat s SQLiteDatabase bez zmeny kodu

Agregace vs Kompozice — Motor (kompozice) vzniká uvnitř Auta a zanikne s ním. Řidič (agregace) existuje nezávisle — auto ho jen „zná", ale nevlastní.
class Motor:
    def __init__(self, typ): self.typ = typ
    def nastartuj(self): print(f"Motor {self.typ}: vrr!")

class Ridic:
    def __init__(self, jmeno): self.jmeno = jmeno

class Auto:
    def __init__(self, znacka, motor_typ, ridic=None):
        self.znacka = znacka
        self.motor = Motor(motor_typ)  # KOMPOZICE: motor vytvoren uvnitr
        self.ridic = ridic             # AGREGACE: predany zvenci

ridic = Ridic("Alice")
auto1 = Auto("Skoda", "1.6 TSI", ridic)
auto2 = Auto("BMW", "2.0 TDI", ridic)   # stejny ridic muze ridit vice aut

auto1.motor.nastartuj()
print(f"{auto1.znacka} ridi {auto1.ridic.jmeno}")

del auto1        # motor zanikne s autem (kompozice)
print(ridic.jmeno)  # "Alice" — ridic stale existuje (agregace)

S — Single Responsibility Principle (SRP)

Každá třída by měla mít pouze jeden důvod ke změně. Pokud třída řeší více věcí najednou (business logika + ukládání + výstup), je těžko testovatelná a změna jedné části ovlivní ostatní.

❌ Špatně — třída Student dělá tři věci najednou: drží data, ukládá se a tiskne report. ✅ Správně — každá zodpovědnost má vlastní třídu.
# ❌ SRP poruseno — Student toho dela prilis
class StudentSpatne:
    def __init__(self, jmeno, znamky):
        self.jmeno = jmeno
        self.znamky = znamky
    def uloz_do_db(self): ...        # zodpovednost 1: databaze
    def tisk_report(self): ...       # zodpovednost 2: output
    def vypocitej_prumer(self): ...  # zodpovednost 3: vypocty

# ✅ SRP dodrzeno — kazda trida jednu zodpovednost
class Student:
    def __init__(self, jmeno, znamky):
        self.jmeno = jmeno
        self.znamky = znamky
    def prumer(self):
        return sum(self.znamky) / len(self.znamky)

class StudentRepository:
    def uloz(self, student): ...     # jen databaze
    def nacti(self, jmeno): ...

class StudentReport:
    def tiskni(self, student):       # jen vystup
        print(f"{student.jmeno}: prumer {student.prumer():.1f}")

O — Open/Closed Principle (OCP)

Kód by měl být otevřen pro rozšíření (nová funkčnost přidáním nové třídy), ale uzavřen pro modifikaci (existující kód měnit co nejméně). Místo if/elif použij polymorfismus.

❌ Každý nový formát exportu vyžaduje úpravu třídy. ✅ Nový formát = nová třída, existující kód beze změny.
# ❌ OCP poruseno — pri pridani noveho formatu upravujes ExporterSpatne
class ExporterSpatne:
    def exportuj(self, data, format):
        if format == "csv": ...
        elif format == "json": ...
        elif format == "xml": ...   # kazde pridani = zmena teto tridy

# ✅ OCP dodrzeno
class Exporter(ABC):
    @abstractmethod
    def exportuj(self, data) -> str: pass

class CSVExporter(Exporter):
    def exportuj(self, data): return ",".join(str(x) for x in data)

class JSONExporter(Exporter):
    def exportuj(self, data):
        import json; return json.dumps(data)

# Novy format = nova trida, ExportService se nemeni
class ExportService:
    def __init__(self, exporter: Exporter):
        self.exporter = exporter
    def zpracuj(self, data):
        return self.exporter.exportuj(data)

L — Liskov Substitution Principle (LSP)

Pokud je B potomkem A, pak všude kde se používá A, lze použít B bez poruchy chování. Potomek nesmí zpřísnit podmínky ani zrušit chování rodiče.

Klasický příklad porušení: Ctverec extends Obdelnik — čtverec nemůže mít nezávisle nastavitelné strany, takže se chová jinak než obdélník.
# ❌ LSP poruseno — Ctverec nespravne dedi od Obdelniku
class ObdelnikSpatne:
    def set_sirka(self, s): self.sirka = s
    def set_vyska(self, v): self.vyska = v
    def plocha(self): return self.sirka * self.vyska

class CtverecSpatne(ObdelnikSpatne):
    def set_sirka(self, s):       # porusuje LSP: nastavuje OBE strany
        self.sirka = s
        self.vyska = s

# Test: ocekavame ze set_sirka a set_vyska jsou nezavisle
r = ObdelnikSpatne(); r.set_sirka(4); r.set_vyska(5)
# Pro CtverecSpatne by plocha = 25 (ne 20) — surprising behavior!

# ✅ LSP spravne — oddelene abstrakce
class Tvar(ABC):
    @abstractmethod
    def plocha(self): pass

class Obdelnik(Tvar):
    def __init__(self, s, v): self.sirka, self.vyska = s, v
    def plocha(self): return self.sirka * self.vyska

class Ctverec(Tvar):           # vlastni hierachie, nezdedi od Obdelnik
    def __init__(self, a): self.a = a
    def plocha(self): return self.a ** 2

I — Interface Segregation Principle (ISP)

Klienti by neměli být nuceni záviset na metodách, které nepoužívají. Raději více malých rozhraní než jedno velké „super-rozhraní".

# ❌ ISP poruseno — Tiskarna musi implementovat i fax a skener
class MultifunkcniZarizeniSpatne(ABC):
    @abstractmethod
    def tiskni(self): pass
    @abstractmethod
    def skenuj(self): pass
    @abstractmethod
    def faxuj(self): pass

class StavaTiskarna(MultifunkcniZarizeniSpatne):
    def tiskni(self): print("Tisknu")
    def skenuj(self): raise NotImplementedError   # ❌ vynucene!
    def faxuj(self):  raise NotImplementedError   # ❌ vynucene!

# ✅ ISP spravne — male oddelene interfacy
class Tiskatelny(ABC):
    @abstractmethod
    def tiskni(self): pass

class Skenovatelny(ABC):
    @abstractmethod
    def skenuj(self): pass

class JenTiskarna(Tiskatelny):       # implementuje jen to, co umi
    def tiskni(self): print("Tisknu")

class MultiFunkce(Tiskatelny, Skenovatelny):   # kombinuje dle potreby
    def tiskni(self): print("Tisknu")
    def skenuj(self): print("Skenuji")

D — Dependency Inversion Principle (DIP)

Vysokoúrovňové moduly by neměly záviset na nízkoúrovňových. Oba by měli záviset na abstrakcích. Abstrakce by neměly záviset na detailech — detaily závisí na abstrakcích.

Prakticky: injektujte závislosti přes konstruktor (Dependency Injection), ne vytvářejte instance přímo uvnitř třídy.
# ❌ DIP poruseno — vysokourovnovy modul zavisi na konkretni DB
class ObjednavkaService:
    def __init__(self):
        self.db = MySQLDatabase()    # ❌ primo zavislost na MySQL

# ✅ DIP spravne — zavisi na abstrakci
class UlozisteSpatne(ABC):
    @abstractmethod
    def uloz(self, data): pass
    @abstractmethod
    def nacti(self, id): pass

class MySQLUloziste(UlozisteSpatne):
    def uloz(self, data): print(f"MySQL: ukladam {data}")
    def nacti(self, id):  return {"id": id, "zdroj": "MySQL"}

class InMemoryUloziste(UlozisteSpatne):  # pro testy
    def __init__(self): self._data = {}
    def uloz(self, data): self._data[data["id"]] = data
    def nacti(self, id):  return self._data.get(id)

class ObjednavkaService:
    def __init__(self, uloziste: UlozisteSpatne):  # ✅ abstrakce
        self.uloziste = uloziste   # Dependency Injection

    def vytvor(self, objednavka):
        self.uloziste.uloz(objednavka)

# Produkce:
svc = ObjednavkaService(MySQLUloziste())
# Testy:
svc_test = ObjednavkaService(InMemoryUloziste())

Shrnutí — co si zapamatovat

4 pilíře OOP:
  • Zapouzdření — soukromá data, veřejné metody. „Vím co umíš, ne jak to děláš uvnitř."
  • Dědičnost — potomek je specializací rodiče (IS-A). Vždy volej super().__init__().
  • Polymorfismus — stejné volání, různý výsledek. Základ pro generický kód.
  • Abstrakce — skryj složitost za rozhraní. Abstraktní třídy = sdílená implementace + kontrakt.
Vztahy: IS-A = dědičnost  |  HAS-A slabé = agregace (◇)  |  HAS-A silné = kompozice (◆)
SOLID: S = jedna zodpovědnost  |  O = rozšiřuj, neměň  |  L = potomek neporuší chování  |  I = malá rozhraní  |  D = závisej na abstrakci

Ukázkový kód — kompletní příklad

Zapouzdření — BankovniUcet skrývá zůstatek (__zustatek = name mangling). Přístup jen přes metody vloz(), vyber() a property zustatek.
class BankovniUcet:
    """Zapouzdreni - skryti stavu"""
    def __init__(self, majitel, zustatek=0):
        self._majitel = majitel
        self.__zustatek = zustatek   # name mangling

    @property
    def zustatek(self):
        return self.__zustatek

    def vloz(self, castka):
        if castka <= 0:
            raise ValueError("Castka musi byt kladna!")
        self.__zustatek += castka

    def vyber(self, castka):
        if castka > self.__zustatek:
            raise ValueError("Nedostatek prostredku!")
        self.__zustatek -= castka

ucet = BankovniUcet("Alice", 1000)
ucet.vloz(500)
ucet.vyber(200)
print(f"Zustatek: {ucet.zustatek} Kc")  # 1300
Polymorfismus — abstraktní třída Zivocich definuje zvuk(). Každý potomek ji implementuje jinak. Funkce udelej_zvuk() funguje s libovolným zvířetem.
from abc import ABC, abstractmethod

class Zivocich(ABC):
    def __init__(self, jmeno):
        self.jmeno = jmeno

    @abstractmethod
    def zvuk(self): pass

    def __str__(self):
        return f"{self.jmeno} rika: {self.zvuk()}"

class Pes(Zivocich):
    def zvuk(self): return "Haf haf!"

class Kocka(Zivocich):
    def zvuk(self): return "Mnau!"

class Had(Zivocich):
    def zvuk(self): return "Ssss!"

# Polymorfismus - stejna funkce, ruzne chovani
def udelej_zvuk(zivocich):
    print(f"  {zivocich}")

for z in [Pes("Rex"), Kocka("Micka"), Had("Ssisko")]:
    udelej_zvuk(z)
Agregace vs Kompozice — Motor (kompozice) je vytvořen uvnitř Auta a zanikne s ním. Řidič (agregace) existuje nezávisle na Autě.
class Motor:
    def __init__(self, typ): self.typ = typ
    def nastartuj(self): print(f"    Motor {self.typ} nastartovan")

class Ridic:
    def __init__(self, jmeno): self.jmeno = jmeno

class Auto:
    def __init__(self, znacka, motor_typ, ridic=None):
        self.znacka = znacka
        self.motor = Motor(motor_typ)   # KOMPOZICE - motor vytvoren uvnitr
        self.ridic = ridic              # AGREGACE - ridic existuje nezavisle

ridic = Ridic("Alice")
auto = Auto("Skoda", "1.6 TSI", ridic)
auto.motor.nastartuj()
print(f"  {auto.znacka} ridi {auto.ridic.jmeno}")
# Kdyz auto zanikne, motor zanikne taky (kompozice)
# Ridic ale existuje dal (agregace)
SOLID — SRP + OCP + DIP. Student má jednu zodpovědnost (data), StudentRepository druhou (ukládání). UkladaniStrategy demonstruje Dependency Inversion — ukládáme přes abstrakci.
# SRP - kazda trida jednu zodpovednost
class Student:
    def __init__(self, jmeno, vek):
        self.jmeno = jmeno
        self.vek = vek

class StudentRepository:
    """Oddelena zodpovednost: uloziste"""
    def __init__(self):
        self._studenti = []

    def pridej(self, student):
        self._studenti.append(student)

    def najdi(self, jmeno):
        return next((s for s in self._studenti if s.jmeno == jmeno), None)

# DIP - zavisej na abstrakci
class UkladaniStrategy(ABC):
    @abstractmethod
    def uloz(self, data): pass

class SouborUkladani(UkladaniStrategy):
    def uloz(self, data):
        print(f"  Ukladam do souboru: {data}")

class DBUkladani(UkladaniStrategy):
    def uloz(self, data):
        print(f"  Ukladam do DB: {data}")

class Aplikace:
    def __init__(self, ukladani: UkladaniStrategy):
        self.ukladani = ukladani   # zavisi na abstrakci, ne na implementaci

    def zpracuj(self, data):
        self.ukladani.uloz(data)

# Snadno zamenitelne
app = Aplikace(SouborUkladani())
app.zpracuj("data1")
app = Aplikace(DBUkladani())
app.zpracuj("data2")
