06 – Asymptotická složitost algoritmu

📐 ČÁST 1 — Notace a přehled složitostí

Co je asymptotická složitost?

Popisuje jak roste čas nebo paměť algoritmu s velikostí vstupu n. Nezáleží na konkrétním hardware — jde o třídu růstu.

Příklad: pro n=10 trvá 1ms, pro n=100 trvá 100ms → složitost O(n²) (100× větší vstup = 100² = 10000× více práce).

🔍 ČÁST 2 — Detailní příklady algoritmů

O(1) — Konstantní

Čas nezávisí na velikosti vstupu. Vždy stejný počet operací.

Přístup k prvku pole, push/pop zásobníku, operace s hash tabulkou.
def get_first(arr):
    return arr[0]        # vždy 1 operace, nezávisle na délce arr

def is_even(n):
    return n % 2 == 0    # vždy 1 operace

O(log n) — Logaritmická

S každým krokem se vstup zmenší na polovinu. Pro n=1000: 1000→500→250→...→1 = cca 10 kroků.

Binární vyhledávání — funguje pouze na seřazeném poli. Vždy půlí prohledávaný interval.
def binarni_hledani(arr, cil):
    levo, pravo = 0, len(arr) - 1
    while levo <= pravo:
        stred = (levo + pravo) // 2
        if arr[stred] == cil:
            return stred
        elif arr[stred] < cil:
            levo = stred + 1    # zahodíme levou půlku
        else:
            pravo = stred - 1   # zahodíme pravou půlku
    return -1

# Pro n=1000: max 10 iterací (log₂(1000) ≈ 10)

O(n) — Lineární

Algoritmus projde každý prvek jednou.

Lineární vyhledávání, součet pole, kopírování, hledání maxima/minima.
def soucet(arr):
    total = 0
    for x in arr:           # n iterací, každá O(1)
        total += x
    return total

def maximum(arr):
    max_val = arr[0]
    for x in arr:           # n iterací
        if x > max_val:
            max_val = x
    return max_val

O(n log n) — Linearitmická

Typická pro efektivní řadící algoritmy — rozděluj a panuj (log n úrovní × n práce na každé úrovni).

Merge Sort: rekurzivně dělí pole na poloviny, pak je slučuje (merge). log n úrovní dělení, každá úroveň = n operací sloučení.
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    stred = len(arr) // 2
    leva = merge_sort(arr[:stred])    # levá půlka
    prava = merge_sort(arr[stred:])   # pravá půlka
    return merge(leva, prava)         # sloučení O(n)

def merge(leva, prava):
    vysledek = []
    i = j = 0
    while i < len(leva) and j < len(prava):
        if leva[i] <= prava[j]:
            vysledek.append(leva[i]); i += 1
        else:
            vysledek.append(prava[j]); j += 1
    return vysledek + leva[i:] + prava[j:]

O(n²) — Kvadratická

Typické pro dva vnořené cykly přes stejný vstup.

Bubble Sort: celkem operací = (n-1)+(n-2)+...+1 = n(n-1)/2 ≈ n²
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):              # vnější: n
        for j in range(n - i - 1): # vnitřní: n-1, n-2, ...
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]

# Naivní hledání duplikátů — O(n²)
def ma_duplikat_pomale(arr):
    for i in range(len(arr)):
        for j in range(i + 1, len(arr)):
            if arr[i] == arr[j]:
                return True
    return False

# Rychlá verze pomocí množiny — O(n)
def ma_duplikat_rychle(arr):
    return len(arr) != len(set(arr))

O(2ⁿ) — Exponenciální

Každý krok zdvojí počet podproblémů. Rychle nepraktické.

Naivní Fibonacci: fib(5) volá fib(4) a fib(3), fib(4) volá fib(3) a fib(2) — obrovské množství duplicitních výpočtů.
# Naivní — O(2^n): fib(30) = ~1 miliarda operací
def fib_pomale(n):
    if n <= 1:
        return n
    return fib_pomale(n-1) + fib_pomale(n-2)  # 2 volání na každé n

# Oprava: memoizace → O(n), každé n se spočítá jen jednou
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_rychle(n):
    if n <= 1:
        return n
    return fib_rychle(n-1) + fib_rychle(n-2)
Memoizace = ukládání výsledků = Space-Time Tradeoff: čas z O(2ⁿ) → O(n), ale potřebujeme O(n) paměti navíc.

📏 ČÁST 3 — Pravidla, paměť, tradeoffs

Pravidla pro určení složitosti z kódu

  1. Konstanty se ignorují: O(3n + 7) = O(n)
  2. Dominantní člen vyhrává: O(n² + n + 1) = O(n²)
  3. Vnořené cykly → násobení: 
    for i in range(n):       # n
    for j in range(n):   # n
        ...              # → O(n²)
  4. Sekvenční bloky → sčítání, pak dominantní člen: 
    for i in range(n): ...          # O(n)
for i in range(n):              # O(n²)
    for j in range(n): ...
# Celkem: O(n) + O(n²) = O(n²)
  5. Větvení → maximum: 
    if podminka:   # O(n)
else:          # O(n²)
# → O(n²)
  6. Rekurze — Master theorem:
    • T(n) = 2·T(n/2) + O(n) → O(n log n) (Merge Sort)
    • T(n) = T(n/2) + O(1) → O(log n) (Binární vyhledávání)
    • T(n) = 2·T(n-1) + O(1) → O(2ⁿ) (Naivní Fibonacci)

Amortizovaná složitost

Průměrná složitost jedné operace přes sérii operací — i když občas je operace drahá.

Dynamické pole (ArrayList / Python list): při přidání prvku obvykle O(1), ale při plném poli → zdvojení kapacity = kopírování = O(n).
# Schéma zdvojování kapacity:
# vložení 1: O(1)   kapacita: 1→2
# vložení 2: O(2)   zdvojení, kopírování
# vložení 3: O(1)
# vložení 4: O(4)   zdvojení, kopírování
# vložení 5,6,7: O(1)
# vložení 8: O(8)   zdvojení, kopírování
# ...
# Celkem pro n vložení: 1+2+1+4+1+1+1+8+... = O(n)
# Průměr na vložení: O(n)/n = O(1) amortizovaně
