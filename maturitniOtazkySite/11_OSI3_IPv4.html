IPv4 adresa

Základní vlastnosti

  • 32 bitů (4 bajty), zapsáno v tečkové desítkové notaci: 192.168.1.10
  • Adresní prostor: 2³² = ~4,3 miliardy adres
  • Skládá se z síťové části a hostitelské části

Speciální adresy

  • 127.0.0.0/8 – loopback (localhost)
  • 169.254.0.0/16 – APIPA (automatická při selhání DHCP)
  • 0.0.0.0 – výchozí cesta / nespecifikovaná adresa
  • 255.255.255.255 – omezený broadcast

Počet hostů = 2^(32 - prefix) - 2 (síťová adresa + broadcast)

Podsíťování (Subnetting)

Příklad: Rozděl 192.168.1.0/24 na 4 podsítě

Potřebujeme 4 podsítě → 2² = 4 → potřeba 2 bity navíc → /26

Podsíť 1: 192.168.1.0/26    (0–63)     Rozsah hostů: .1 – .62,   Broadcast: .63
Podsíť 2: 192.168.1.64/26   (64–127)   Rozsah hostů: .65 – .126, Broadcast: .127
Podsíť 3: 192.168.1.128/26  (128–191)  Rozsah hostů: .129 – .190, Broadcast: .191
Podsíť 4: 192.168.1.192/26  (192–255)  Rozsah hostů: .193 – .254, Broadcast: .255

Každá podsíť má 62 použitelných adres.

Postup podsíťování

  1. Zjistit kolik podsítí/hostů potřebujeme
  2. Vypočítat nový prefix: pro N podsítí potřebujeme ⌈log₂(N)⌉ bitů navíc
  3. Nebo pro H hostů: prefix = 32 - ⌈log₂(H+2)⌉
  4. Spočítat rozsahy: velikost bloku = 2^(32-prefix)

VLSM (Variable Length Subnet Masking)

Umožňuje používat různé velikosti podsítí z jednoho adresního bloku. Přiřazujeme od největší podsítě po nejmenší.

Fragmentace (IPv4)

Proč fragmentace

Když je paket větší než MTU (Maximum Transmission Unit) odchozího spoje (typicky 1500 B Ethernet), musí být rozdělen na menší fragmenty.

Fragmentační pole v hlavičce

  • Identification – společné ID pro všechny fragmenty jednoho paketu
  • MF (More Fragments) – 1 = následují další fragmenty, 0 = poslední fragment
  • DF (Don't Fragment) – 1 = nefragmentovat (při překročení MTU vrátí ICMP „Fragmentation Needed")
  • Fragment Offset – pozice fragmentu (v 8B jednotkách)

Příklad fragmentace

Paket: 4000 B dat, MTU = 1500 B

Fragment 1: Offset=0,    MF=1, Data=1480 B
Fragment 2: Offset=185,  MF=1, Data=1480 B    (185 × 8 = 1480)
Fragment 3: Offset=370,  MF=0, Data=1040 B    (370 × 8 = 2960)

Celkem: 1480 + 1480 + 1040 = 4000 B

Path MTU Discovery

Mechanismus pro zjištění nejmenšího MTU na cestě. Odesílatel nastaví DF=1, pokud router nemůže přeposlat vrátí ICMP Type 3 Code 4 (Destination Unreachable, Fragmentation Needed) s MTU. Odesílatel zmenší pakety.

Poznámka: Fragmentace snižuje výkon (režie na přeskládání) a zvyšuje riziko ztráty dat (ztráta jednoho fragmentu = ztráta celého paketu). Proto je preferováno Path MTU Discovery.
