15 – OSI 4 – Transportní vrstva – UDP, TCP, segmentace, řízení rychlosti, kontrola chyb

Role transportní vrstvy

  • Logická komunikace mezi procesy (aplikacemi) na různých hostech
  • Segmentace a znovusestavení dat
  • Multiplexing – více aplikací na jednom hostu (identifikace přes porty)
  • Dva hlavní protokoly: TCP (spolehlivý) a UDP (nespolehlivý)

Socket

Unikátní identifikace spojení: IP adresa + port. Např. 192.168.1.10:443.

Plné TCP spojení je identifikováno čtveřicí: (src_IP, src_port, dst_IP, dst_port)

UDP (User Datagram Protocol)

Vlastnosti

  • Nespolehlivý – žádné potvrzení doručení
  • Bezspojový (connectionless) – žádný handshake
  • Nízká režie – hlavička pouze 8 bajtů
  • Rychlý – vhodný pro real-time aplikace
  • Negarantuje pořadí doručení

TCP (Transmission Control Protocol)

Vlastnosti

  • Spolehlivý – potvrzení doručení (ACK)
  • Spojově orientovaný – 3-way handshake
  • Garantuje pořadí a úplnost dat
  • Řízení toku a přetížení
  • Hlavička 20–60 bajtů

3-Way Handshake (navázání spojení)

Klient                         Server
  |                               |
  |---- SYN (seq=100) ----------->|   1. Klient chce spojení
  |                               |
  |<--- SYN+ACK (seq=300,ack=101)-|   2. Server potvrzuje + nabízí
  |                               |
  |---- ACK (seq=101,ack=301) --->|   3. Klient potvrzuje
  |                               |
  |===== SPOJENÍ NAVÁZÁNO =======|

4-Way Termination (ukončení spojení)

Klient                         Server
  |                               |
  |---- FIN ---------------------->|   1. Klient chce ukončit
  |<--- ACK ----------------------|   2. Server potvrzuje
  |                               |
  |<--- FIN ----------------------|   3. Server chce ukončit
  |---- ACK ---------------------->|   4. Klient potvrzuje
  |                               |
  |==== SPOJENÍ UKONČENO ========|

Segmentace a znovusestavení

Segmentace

  • Data z aplikace jsou rozdělena na segmenty velikosti ≤ MSS (Maximum Segment Size)
  • MSS = MTU − IP hlavička − TCP hlavička = 1500 − 20 − 20 = 1460 B (typicky)
  • Každý segment má sequence number = pozice prvního bajtu v proudu

Znovusestavení

  • Příjemce řadí segmenty podle sequence number
  • Potvrzuje přijetí pomocí ACK number (další očekávaný bajt)
  • Chybějící segmenty → vyžádání opětovného odeslání

Řízení přenosové rychlosti

Flow Control (řízení toku)

Příjemce reguluje rychlost odesílatele pomocí Window Size.

Window Size = 4000 B → odesílatel může poslat max 4000 B bez ACK

Host A ---[Seg 1 (1000B)]---> Host B
Host A ---[Seg 2 (1000B)]---> Host B
Host A ---[Seg 3 (1000B)]---> Host B
Host A ---[Seg 4 (1000B)]---> Host B
Host A <--[ACK, Window=2000]- Host B   ← příjemce zpomaluje
Host A ---[Seg 5 (1000B)]---> Host B
Host A ---[Seg 6 (1000B)]---> Host B
Host A <--[ACK, Window=0]---- Host B   ← STOP! Buffer plný

Congestion Control (řízení přetížení)

Odesílatel reguluje rychlost sám podle stavu sítě:

MechanismusPopis
Slow StartZačni s cwnd=1 MSS, zdvojnásob po každém ACK (exponenciální růst)
Congestion AvoidancePo dosažení ssthresh: cwnd += 1 MSS za RTT (lineární růst)
Fast Retransmit3 duplikátní ACK → okamžitý retransmit bez čekání na timeout
Fast RecoveryPo fast retransmit: ssthresh = cwnd/2, nepřecházet na slow start

Kontrola chyb

  • Checksum – ověření integrity segmentu (povinný v TCP i UDP/IPv6)
  • Sequence Numbers – detekce ztráty, duplicity a špatného pořadí
  • ACK – potvrzení = odeslání čísla dalšího očekávaného bajtu
  • Retransmission Timer – pokud nepřijde ACK do timeoutu → opětovné odeslání
  • Selective ACK (SACK) – potvrzení konkrétních přijatých rozsahů (TCP option)
