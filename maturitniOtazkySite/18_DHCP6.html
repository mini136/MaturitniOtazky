SLAAC (Stateless Address Autoconfiguration)

  1. Klient pošle Router Solicitation (RS) na ff02::2
  2. Router odpoví Router Advertisement (RA) s prefixem (typicky /64)
  3. Klient vytvoří adresu: prefix + Interface ID (EUI-64 nebo random)
  4. Klient provede DAD (Duplicate Address Detection)
  5. Link-local adresa (fe80::) se vytvoří automaticky vždy

Stateless DHCPv6

Adresa přes SLAAC, ale DNS, NTP a další parametry z DHCPv6 serveru.

Konfigurace – Cisco router

! Router nastaví O flag
interface GigabitEthernet0/0
  ipv6 address 2001:db8:1::1/64
  ipv6 nd other-config-flag              ! O=1 → kontaktuj DHCPv6 pro DNS
  no shutdown

! DHCPv6 pool (stateless – žádné adresy, jen parametry)
ipv6 dhcp pool STATELESS_POOL
  dns-server 2001:4860:4860::8888       ! Google DNS
  domain-name firma.local

interface GigabitEthernet0/0
  ipv6 dhcp server STATELESS_POOL

Stateful DHCPv6

Server přiděluje plnou IPv6 adresu i další parametry. Ekvivalent DHCPv4.

Proces SARR (Stateful DHCPv6)

Klient                                 Server
  |                                       |
  |-- SOLICIT (multicast ff02::1:2) ----->|  1. "Hledám DHCPv6 server"
  |                                       |     (UDP port 547)
  |<-------------- ADVERTISE -------------|  2. "Jsem tu, nabízím..."
  |                                       |
  |-- REQUEST --------------------------->|  3. "Chci tuto adresu"
  |                                       |
  |<-------------- REPLY -----------------|  4. "OK, je tvoje"
  |                                       |

Konfigurace – Cisco router (Stateful)

! Router nastaví M flag
interface GigabitEthernet0/0
  ipv6 address 2001:db8:1::1/64
  ipv6 nd managed-config-flag            ! M=1 → adresa z DHCPv6
  ipv6 nd prefix 2001:db8:1::/64 no-autoconfig  ! zakáže SLAAC
  no shutdown

! DHCPv6 pool (stateful – přiděluje adresy)
ipv6 dhcp pool STATEFUL_POOL
  address prefix 2001:db8:1::/64 lifetime 86400 43200
  dns-server 2001:4860:4860::8888
  domain-name firma.local

interface GigabitEthernet0/0
  ipv6 dhcp server STATEFUL_POOL

DHCPv6 Prefix Delegation (PD)

ISP přidělí zákazníkovi celý prefix (např. /48 nebo /56), zákazníkův router si pak z něj vytváří podsítě /64.

! ISP router (delegating router)
ipv6 dhcp pool ISP_PD
  prefix-delegation pool CUSTOMER_PREFIXES

ipv6 local pool CUSTOMER_PREFIXES 2001:db8::/32 48

! Zákaznický router (requesting router)
interface GigabitEthernet0/0
  ipv6 dhcp client pd MY_PREFIX         ! získej prefix od ISP

interface GigabitEthernet0/1
  ipv6 address MY_PREFIX ::1/64         ! použij přidělený prefix na LAN

DHCPv6 Relay

! Relay na routeru – přeposílání DHCPv6 zpráv
interface GigabitEthernet0/0
  ipv6 dhcp relay destination 2001:db8:10::5    ! DHCPv6 server
