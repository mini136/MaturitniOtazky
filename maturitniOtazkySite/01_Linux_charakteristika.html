01 – OS Linux – Charakteristika, struktura OS, start systému, služby, procesy, paměť, balíčkovací systém

Charakteristika Linuxu

  • Open-source operační systém založený na UNIXu
  • Vytvořil Linus Torvalds v roce 1991 (jádro)
  • Multiplatformní – běží na x86, ARM, MIPS, RISC-V…
  • Multitasking – souběžné provádění více procesů
  • Multiuser – více uživatelů současně
  • Licencován pod GPL (GNU General Public License)
  • Distribuce: Ubuntu, Debian, Fedora, CentOS/RHEL, Arch Linux, openSUSE

Struktura operačního systému

Vrstvy OS

VrstvaPopis
HardwareFyzické komponenty (CPU, RAM, disky, síťové karty)
Jádro (Kernel)Správa HW, procesy, paměť, I/O, souborové systémy
Systémové knihovnyglibc, libstdc++ – rozhraní mezi aplikacemi a jádrem
Systémové nástrojeShell (bash, zsh), coreutils, systemd
Uživatelské aplikaceTextové editory, prohlížeče, servery (Apache, Nginx)

Typy jádra

  • Monolitické jádro – Linux – vše v jednom prostoru jádra (s modulární podporou)
  • Mikrojádro – MINIX, QNX – minimální jádro, služby v user space
  • Hybridní – Windows NT, macOS

Kernel moduly

lsmod              # seznam načtených modulů
modprobe modul     # načtení modulu
rmmod modul        # odebrání modulu
modinfo modul      # informace o modulu

Start systému (Boot process)

Průběh startu

  1. BIOS/UEFI – POST (Power-On Self Test), inicializace HW, hledání bootloaderu
  2. Bootloader (GRUB2) – načte jádro a initramfs do RAM 
    # Konfigurace: /etc/default/grub
# Aktualizace: grub-mkconfig -o /boot/grub/grub.cfg
# nebo: update-grub (Debian/Ubuntu)
  3. Jádro (Kernel) – inicializace HW, připojení root FS, spuštění init procesu
  4. initramfs/initrd – dočasný souborový systém v RAM pro načtení ovladačů
  5. Init systém (systemd/SysVinit) – spuštění služeb a cílů (targets)
  6. Login prompt – přihlašovací obrazovka (getty / display manager)

Služby (Services)

Spravovány přes systemd (aktuálně nejpoužívanější init systém).

systemctl start sshd        # spuštění služby
systemctl stop sshd         # zastavení
systemctl restart sshd      # restart
systemctl enable sshd       # automatický start při bootu
systemctl disable sshd      # zakázání autostartu
systemctl status sshd       # stav služby
systemctl list-units --type=service  # seznam služeb

Unit soubory

Umístění: /etc/systemd/system/ nebo /lib/systemd/system/

[Unit]
Description=Moje služba
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/muj-program
Restart=on-failure

[Install]
WantedBy=multi-user.target

Procesy

Základní pojmy

  • PID – Process ID – unikátní identifikátor procesu
  • PPID – Parent PID – ID rodičovského procesu
  • Init (PID 1) – první proces (systemd)
  • Zombie – ukončený proces, jehož rodič si nevyzvedl exit kód
  • Orphan – proces, jehož rodič skončil → adoptován init procesem
  • Daemon – služba běžící na pozadí

Stavy procesů

StavOznačeníPopis
RunningRPrávě se provádí na CPU
SleepingSČeká na událost (přerušitelný)
Disk SleepDNepřerušitelný spánek (I/O)
StoppedTPozastavený (SIGSTOP)
ZombieZUkončený, čeká na wait()

Práce s procesy

ps aux                  # výpis všech procesů
ps -ef                  # výpis s hierarchií
top / htop              # interaktivní monitor
pstree                  # stromová struktura procesů
kill PID                # SIGTERM (15) – požádá o ukončení
kill -9 PID             # SIGKILL (9) – vynucené ukončení
kill -STOP PID          # pozastavení
kill -CONT PID          # obnovení
nice -n 10 program      # spuštění s nižší prioritou
renice -n 5 -p PID      # změna priority
jobs                    # úlohy na pozadí
bg / fg                 # pozadí / popředí
nohup cmd &             # příkaz přežije odhlášení

Paměť

Typy paměti

  • Fyzická paměť (RAM) – skutečná fyzická paměť
  • Virtuální paměť – abstrakce nad fyzickou pamětí, pro každý proces zvlášť
  • Swap – odkládací prostor na disku, když dochází RAM
  • Cache/Buffer – kernel využívá volnou RAM pro cache diskových operací

Správa paměti

free -h                 # přehled paměti (human-readable)
cat /proc/meminfo       # detailní info o paměti
vmstat                  # statistika virtuální paměti
swapon -s               # swap informace
swapon /dev/sdXn        # zapnutí swapu
swapoff /dev/sdXn       # vypnutí swapu

OOM Killer

Out-Of-Memory Killer – mechanismus jádra, který při nedostatku paměti zabije proces s nejvyšším „OOM score", aby systém přežil.

# Nastavení OOM score adjustmentu
echo -1000 > /proc/PID/oom_score_adj   # ochrana procesu
echo 1000 > /proc/PID/oom_score_adj    # priorita zabití

Balíčkovací systém

Debian/Ubuntu – APT (dpkg)

apt update              # aktualizace seznamu balíčků
apt upgrade             # upgrade instalovaných balíčků
apt install balicek     # instalace
apt remove balicek      # odinstalace
apt search balicek      # hledání
dpkg -i balicek.deb     # ruční instalace .deb
dpkg -l                 # seznam instalovaných

RHEL/CentOS/Fedora – DNF (rpm)

dnf update              # aktualizace
dnf install balicek     # instalace
dnf remove balicek      # odinstalace
dnf search balicek      # hledání
rpm -ivh balicek.rpm    # ruční instalace .rpm
rpm -qa                 # seznam instalovaných

Arch Linux – pacman

pacman -Syu             # synchronizace + upgrade
pacman -S balicek       # instalace
pacman -R balicek       # odinstalace
pacman -Ss balicek      # hledání

Univerzální balíčkovací formáty

  • Snap (Canonical) – sandboxed balíčky, snap install balicek
  • Flatpak – desktopové aplikace, flatpak install balicek
  • AppImage – jeden spustitelný soubor, žádná instalace
