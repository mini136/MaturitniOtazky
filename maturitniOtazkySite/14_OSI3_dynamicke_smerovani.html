14 – OSI 3 – Dynamické směrování (RIP, OSPF), konfigurace, metriky, AD

Dynamické směrovací protokoly

Routery si automaticky vyměňují informace o sítích a hledají nejlepší cesty. Při výpadku spoje se automaticky přepočítají cesty (konvergence).

RIP (Routing Information Protocol)

Vlastnosti

  • Distance Vector, metrika = počet hopů (max 15, 16 = unreachable)
  • Aktualizace každých 30 sekund (celá tabulka)
  • AD = 120
  • Pomalá konvergence

Verze

Ochrana proti smyčkám

  • Split Horizon – nepřeposílej cestu zpět tím rozhraním, odkud jsi ji naučil
  • Route Poisoning – při výpadku nastav metriku na 16 (nedosažitelné)
  • Poison Reverse – oznam nedosažitelnost zpět odesílateli
  • Hold-down Timer – po výpadku čekej, než přijmeš novou cestu (180s)

Konfigurace RIPv2 (Cisco)

router rip
  version 2
  network 192.168.1.0
  network 10.0.0.0
  no auto-summary           ! důležité pro VLSM
  passive-interface Gi0/0   ! neodesílej aktualizace na LAN rozhraní

! Ověření
show ip rip database
show ip route rip

OSPF (Open Shortest Path First)

Vlastnosti

  • Link-State protokol, algoritmus Dijkstra (SPF)
  • AD = 110
  • Metrika = cost (založena na šířce pásma: cost = 10⁸ / bandwidth v bps)
  • Rychlá konvergence, bez smyček
  • Hierarchický – používá oblasti (areas)
  • Podporuje VLSM, autentizaci

Poznámka: Pro rozlišení FastEthernet a GigE je třeba zvýšit referenční bandwidth: auto-cost reference-bandwidth 10000 (v Mbps).

OSPF oblasti (Areas)

  • Area 0 – backbone, všechny ostatní oblasti se musí připojit k area 0
  • Hierarchie snižuje velikost LSDB a zrychluje konvergenci
  • ABR (Area Border Router) – router na hranici dvou oblastí
  • ASBR (AS Boundary Router) – router na hranici OSPF a jiného AS

OSPF operace

  1. Navázání sousedství – Hello pakety (multicast 224.0.0.5/6)
  2. Výměna LSDB – LSA (Link-State Advertisements)
  3. Výpočet SPF – Dijkstrův algoritmus → nejkratší strom
  4. Naplnění směrovací tabulky

DR / BDR (Designated Router)

Na multi-access sítích (Ethernet) se volí DR a BDR pro redukci LSA trafficu:

  • Routery posílají LSA na 224.0.0.6 (All DR Routers)
  • DR distribuuje na 224.0.0.5 (All OSPF Routers)
  • Volba dle: 1. Nejvyšší priority (0 = nevolen), 2. Nejvyšší Router ID

Konfigurace OSPF (Cisco)

router ospf 1                               ! process ID (lokální)
  router-id 1.1.1.1                         ! manuální Router ID
  network 192.168.1.0 0.0.0.255 area 0     ! wildcard maska!
  network 10.0.0.0 0.0.0.3 area 1
  passive-interface GigabitEthernet0/0      ! neodesílej Hello na LAN
  auto-cost reference-bandwidth 10000       ! pro rozlišení GigE

interface GigabitEthernet0/0
  ip ospf priority 100                      ! ovlivnění volby DR
  ip ospf cost 10                           ! ruční nastavení cost

! Ověření
show ip ospf neighbor
show ip ospf interface brief
show ip ospf database
show ip route ospf
