20 – Firewall – Principy, typy, konfigurace pravidel, NAT, DMZ, VPN

Co je Firewall

Síťové bezpečnostní zařízení (HW nebo SW), které filtruje síťový provoz na základě definovaných pravidel. Chrání síť před neoprávněným přístupem.

Typy firewallů

1. Stateless (Packet Filtering) Firewall

  • Filtruje každý paket nezávisle na základě hlaviček (L3/L4)
  • Kontroluje: zdrojová/cílová IP, zdrojový/cílový port, protokol
  • Nesleduje stav spojení
  • Rychlý, ale snadno obcházen (fragmentace, podvržené ACK)
  • Příklad: ACL na Cisco routeru

2. Stateful Firewall

  • Udržuje tabulku stavů spojení (state table)
  • Sleduje celé spojení (SYN → SYN+ACK → ACK...)
  • Automaticky povolí odpovědi na povolené odchozí spojení
  • Bezpečnější – rozpozná nelegitimní pakety
  • Příklad: iptables/nftables s conntrack, Cisco ASA, pfSense

3. Aplikační (Proxy / Application Layer) Firewall

  • Pracuje na vrstvě 7 – rozumí obsahu (HTTP, DNS, FTP...)
  • Může filtrovat URL, blokovat malware, inspekce SSL
  • WAF (Web Application Firewall) – ochrana webových aplikací (SQL injection, XSS)
  • Příklad: ModSecurity, Cloudflare WAF, F5

4. Next-Generation Firewall (NGFW)

  • Kombinace stateful + aplikační inspekce + IPS + DPI
  • Identifikace aplikací (i na nestandardních portech)
  • Příklad: Palo Alto, Fortinet FortiGate, Cisco Firepower

Konfigurace pravidel

ACL na Cisco (stateless)

! Standard ACL (filtruje jen podle zdrojové IP)
access-list 10 permit 192.168.1.0 0.0.0.255
access-list 10 deny any

! Extended ACL (src/dst IP, port, protokol)
access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80     ! povolí HTTP
access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 443    ! povolí HTTPS
access-list 100 permit udp any any eq 53                         ! DNS
access-list 100 deny ip any any                                  ! zamítni zbytek

! Aplikace na rozhraní
interface GigabitEthernet0/0
  ip access-group 100 in        ! in = příchozí provoz na rozhraní

! Named ACL
ip access-list extended WEB_ACL
  permit tcp 192.168.1.0 0.0.0.255 any eq 80
  permit tcp 192.168.1.0 0.0.0.255 any eq 443
  deny ip any any log

iptables (Linux – stateful)

# Výchozí politika – zamítni vše
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# Povol established/related spojení (stateful)
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Povol SSH z konkrétní sítě
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 22 -j ACCEPT

# Povol HTTP a HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Povol ICMP (ping)
iptables -A INPUT -p icmp -j ACCEPT

# Loguj zamítnuté
iptables -A INPUT -j LOG --log-prefix "FIREWALL DROP: "

NAT (Network Address Translation)

Překlad privátních IP adres na veřejné → umožňuje komunikaci vnitřní sítě s internetem.

Typy NAT

TypPopis
Static NAT1:1 mapování (privátní ↔ veřejná). Pro servery.
Dynamic NATPool veřejných adres, přiděluje se dynamicky.
PAT / NAT OverloadVíce privátních → 1 veřejná (rozlišení přes porty). Nejběžnější.

Konfigurace NAT (Cisco)

! PAT (NAT Overload) – nejběžnější
access-list 1 permit 192.168.1.0 0.0.0.255

ip nat inside source list 1 interface GigabitEthernet0/1 overload

interface GigabitEthernet0/0
  ip nat inside                          ! vnitřní rozhraní

interface GigabitEthernet0/1
  ip nat outside                         ! vnější rozhraní (internet)

! Static NAT – mapování serveru
ip nat inside source static 192.168.1.10 203.0.113.10

! Port forwarding (server za NAT)
ip nat inside source static tcp 192.168.1.10 80 203.0.113.1 80

! Ověření
show ip nat translations
show ip nat statistics

DMZ (Demilitarized Zone)

Oddělená síťová zóna mezi vnitřní sítí a internetem. Obsahuje veřejně přístupné servery (web, mail, DNS).

  Internet
     |
  [Firewall]
   /     \
  DMZ    LAN
  |       |
 Web   Klienti
 Mail  Databáze

Pravidla DMZ

  • Internet → DMZ: povoleny specifické služby (HTTP, SMTP)
  • DMZ → LAN: zakázáno (nebo jen konkrétní porty, např. DB)
  • LAN → DMZ: povoleno
  • LAN → Internet: povoleno (přes NAT)
  • DMZ → Internet: omezeno (jen nutné aktualizace)

VPN (Virtual Private Network)

Šifrovaný tunel přes veřejnou síť (internet) → bezpečná komunikace jako na privátní síti.

Typy VPN

TypPopisPoužití
Site-to-SitePropojení dvou sítí (router-router)Pobočky firmy
Remote AccessKlient se připojuje k firemní sítiPráce z domova

VPN protokoly

ProtokolPopis
IPSecL3 tunel, AH (integrita) + ESP (šifrování), IKE pro výměnu klíčů
SSL/TLS VPNPřes HTTPS (port 443), klient nepotřebuje speciální SW
OpenVPNOpen-source, SSL/TLS, UDP/TCP
WireGuardModerní, rychlý, jednoduchý, UDP
PPTPZastaralý, nebezpečný – nepoužívat
L2TP/IPSecL2 tunel + IPSec šifrování
GREGeneric Routing Encapsulation – tunelování (bez šifrování)

IPSec režimy

  • Transport mode – šifruje jen data (payload), IP hlavička zůstává → host-to-host
  • Tunnel mode – šifruje celý původní paket, nová IP hlavička → site-to-site
Shrnutí pro maturitu: Stateless = ACL (L3/L4), Stateful = sleduje spojení, Aplikační = L7 inspekce. NAT Overload (PAT) = nejběžnější. DMZ = zóna pro veřejné servery. VPN = šifrovaný tunel přes internet.
