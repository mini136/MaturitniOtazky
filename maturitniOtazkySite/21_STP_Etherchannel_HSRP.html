21 – STP, EtherChannel a HSRP – Principy, konfigurace, failover, BPDU

STP (Spanning Tree Protocol) – IEEE 802.1D

Problém: Smyčky na L2

  • Redundantní spoje mezi přepínači → broadcastová bouře (broadcast storm)
  • Rámce se posílají dokola → MAC tabulka je nestabilní (MAC flapping)
  • Síť se stane nepoužitelnou

STP eliminuje smyčky tím, že logicky zablokuje redundantní porty a udržuje stromovou topologii.

BPDU (Bridge Protocol Data Unit)

Zprávy, které si přepínače vyměňují pro volbu Root Bridge a určení stavu portů.

  • Posílány na multicast 01:80:C2:00:00:00
  • Obsahují: Bridge ID (priority + MAC), Root Bridge ID, cost, port ID
  • Hello timer = 2 sekundy (default)

Bridge ID

Bridge ID = Priority (4 bity) + Extended System ID (12 bitů) + MAC adresa (48 bitů)
                                  ↑ VLAN ID (pokud PVST+)
Default priority = 32768 (+ VLAN ID)

Volba Root Bridge

Přepínač s nejnižším Bridge ID se stane Root Bridge.

Konfigurace STP (Cisco)

! Nastavení Root Bridge (nízká priorita)
spanning-tree vlan 10 root primary         ! priority 24576
spanning-tree vlan 10 root secondary       ! priority 28672

! Nebo manuálně
spanning-tree vlan 10 priority 4096        ! násobek 4096

! PortFast – přeskočí listening/learning (pro koncové stanice)
interface FastEthernet0/1
  spanning-tree portfast

! BPDU Guard – při přijetí BPDU zablokuj port (ochrana)
interface FastEthernet0/1
  spanning-tree bpduguard enable

! Ověření
show spanning-tree
show spanning-tree vlan 10

EtherChannel

Sdružení více fyzických linek do jedné logické – zvýšení propustnosti + redundance. STP vidí jen jednu linku → žádné blokování.

Konfigurace EtherChannel (LACP)

! Na obou přepínačích
interface range GigabitEthernet0/1-2
  channel-group 1 mode active             ! LACP active
  ! channel-group 1 mode passive          ! LACP passive

! Konfigurace Port-Channel rozhraní
interface Port-Channel1
  switchport mode trunk
  switchport trunk allowed vlan 10,20

HSRP (Hot Standby Router Protocol)

Cisco proprietární protokol pro redundanci výchozí brány. Dva (nebo více) routerů sdílejí virtuální IP adresu.

Princip HSRP

  • Active router – obsluhuje provoz, odpovídá na virtuální IP
  • Standby router – čeká, převezme při výpadku Active
  • Klienti mají jako gateway virtuální IP → failover transparentní
  • Virtuální MAC adresa: 0000.0c07.acXX (XX = group number)

Konfigurace HSRP

! Router 1 (Active)
interface GigabitEthernet0/0
  ip address 192.168.1.2 255.255.255.0
  standby 1 ip 192.168.1.1              ! virtuální IP (gateway pro klienty)
  standby 1 priority 110                 ! vyšší = Active (default 100)
  standby 1 preempt                      ! při obnově převezmi zpět

! Router 2 (Standby)
interface GigabitEthernet0/0
  ip address 192.168.1.3 255.255.255.0
  standby 1 ip 192.168.1.1
  standby 1 priority 90
  standby 1 preempt

! Ověření
show standby
show standby brief
