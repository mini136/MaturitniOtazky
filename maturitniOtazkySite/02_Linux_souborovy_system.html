Žurnálování (Journaling)

Žurnálovací FS (ext3, ext4, XFS) zapisují změny nejprve do žurnálu (logu), teprve pak na disk. Při havárii lze oddíl obnovit z žurnálu → méně poškozených dat.

Formátování disků

# Vytvoření souborového systému
mkfs.ext4 /dev/sdb1        # formátování na ext4
mkfs.xfs /dev/sdb1         # formátování na XFS
mkfs.vfat /dev/sdb1        # FAT32
mkswap /dev/sdb2           # swap oddíl

# Kontrola souborového systému
fsck /dev/sdb1              # kontrola a oprava
e2fsck -f /dev/sdb1         # vynucená kontrola ext FS

Dělení disků (Partitioning)

Typy tabulek oddílů

GPT~9.4 ZB128 (výchozí)UEFI

Nástroje pro dělení

# fdisk – pro MBR i GPT
fdisk /dev/sdb              # interaktivní dělení
fdisk -l                    # výpis oddílů

# gdisk – specificky pro GPT
gdisk /dev/sdb

# parted – universální
parted /dev/sdb
  mklabel gpt               # vytvoření GPT tabulky
  mkpart primary ext4 1MiB 100%  # vytvoření oddílu

# lsblk – přehled blokových zařízení
lsblk                       # stromová struktura disků
lsblk -f                    # včetně souborových systémů

LVM – Logical Volume Manager

Umožňuje flexibilní správu diskového prostoru – snadné zvětšování/zmenšování oddílů.

# Fyzické svazky → Skupiny svazků → Logické svazky
pvcreate /dev/sdb1          # vytvoření Physical Volume
vgcreate moje_vg /dev/sdb1  # vytvoření Volume Group
lvcreate -L 10G -n moje_lv moje_vg  # Logical Volume
mkfs.ext4 /dev/moje_vg/moje_lv      # formátování
lvextend -L +5G /dev/moje_vg/moje_lv # zvětšení

Připojování souborových systémů (Mounting)

Ruční připojení

mount /dev/sdb1 /mnt/data        # připojení
mount -t ntfs /dev/sdc1 /mnt/win # s určením typu FS
umount /mnt/data                  # odpojení
mount -o ro /dev/sdb1 /mnt/data  # jen pro čtení

Automatické připojení – /etc/fstab

# Formát: zařízení   bod_připojení   typ   volby   dump   pass
/dev/sda1       /               ext4    defaults        0       1
/dev/sda2       /home           ext4    defaults        0       2
/dev/sda3       swap            swap    defaults        0       0
UUID=xxxx-xxxx  /mnt/data       ext4    defaults,noatime 0      2
//server/share  /mnt/samba      cifs    credentials=/etc/samba/cred 0 0
UUID – doporučeno místo /dev/sdX, protože UUID se nemění při přidání disků. Zjistíme: blkid 
mount -a                # připojení všeho z /etc/fstab
findmnt                 # zobrazení stromové struktury mountů
df -h                   # využití diskového prostoru
du -sh /var             # velikost adresáře

Speciální soubory v /dev

  • /dev/null – „černá díra" – zahazuje vše co se do něj zapíše
  • /dev/zero – generuje nulové bajty
  • /dev/random – náhodná data (blokující)
  • /dev/urandom – pseudonáhodná data (neblokující)
  • /dev/sda – první SCSI/SATA disk
  • /dev/sda1 – první oddíl prvního disku

Inody (Inodes)

Každý soubor má inode – datovou strukturu obsahující metadata (vlastník, práva, časové značky, ukazatele na datové bloky). Název souboru je uložen v adresáři jako odkaz na inode.

ls -i soubor        # zobrazení čísla inode
df -i               # využití inodů
stat soubor         # detailní info o souboru

Linky

  • Hard link – další název pro stejný inode; nefunguje přes FS, nefunguje na adresáře
  • Symbolic (soft) link – ukazatel na cestu; funguje přes FS i na adresáře
ln soubor hardlink          # hard link
ln -s soubor symlink        # symbolic link
