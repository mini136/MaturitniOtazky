17 – DHCPv4 – Funkce, konfigurace rozsahů, přidělování dle MAC

Co je DHCP

Dynamic Host Configuration Protocol – automaticky přiděluje IP adresu a síťové parametry klientům. Pracuje na aplikační vrstvě, používá UDP porty 67 (server) a 68 (klient).

Co DHCP přiděluje

  • IP adresu
  • Masku podsítě
  • Výchozí bránu (default gateway)
  • DNS server(y)
  • Volitelně: doménové jméno, NTP server, TFTP server, lease time atd.

DORA – Proces přidělení adresy

Klient                                 Server
  |                                       |
  |-- DISCOVER (broadcast) -------------->|  1. "Je tu DHCP server?"
  |   src: 0.0.0.0:68                    |     dst: 255.255.255.255:67
  |                                       |
  |<-------------- OFFER (unicast/bc) ----|  2. "Nabízím ti 192.168.1.10"
  |                                       |
  |-- REQUEST (broadcast) --------------->|  3. "Chci tu adresu 192.168.1.10"
  |                                       |     (broadcast – aby ostatní servery věděly)
  |                                       |
  |<-------------- ACK (unicast/bc) ------|  4. "OK, 192.168.1.10 je tvoje"
  |                                       |
  |===== KONFIGURACE DOKONČENA ==========|

DHCP zprávy

ZprávaSměrPopis
DISCOVERKlient → BroadcastHledání DHCP serveru
OFFERServer → KlientNabídka IP adresy
REQUESTKlient → BroadcastŽádost o konkrétní nabídku
ACKServer → KlientPotvrzení přidělení
NAKServer → KlientZamítnutí (adresa již obsazena)
RELEASEKlient → ServerUvolnění adresy
DECLINEKlient → ServerOdmítnutí (např. IP konflikt)
INFORMKlient → ServerŽádost o parametry (IP mám)

Lease (zápůjčka)

  • IP adresa je přidělena na omezenou dobu (lease time)
  • Při 50 % lease → klient pošle REQUEST pro obnovu (unicast na server)
  • Při 87,5 % lease → klient pošle REQUEST broadcast (jakýkoli DHCP server)
  • Po vypršení → adresa se vrátí do poolu

Konfigurace DHCP serveru

Cisco router jako DHCP server

! Vyloučení adres (gateway, servery)
ip dhcp excluded-address 192.168.1.1 192.168.1.10

! Definice poolu
ip dhcp pool LAN_POOL
  network 192.168.1.0 255.255.255.0       ! rozsah sítě
  default-router 192.168.1.1              ! výchozí brána
  dns-server 8.8.8.8 8.8.4.4             ! DNS servery
  domain-name firma.local                  ! doménové jméno
  lease 7                                  ! lease 7 dní (default: 1 den)

! Ověření
show ip dhcp binding                       ! přidělené adresy
show ip dhcp pool                          ! statistiky poolu
show ip dhcp conflict                      ! konflikty

Rezervace dle MAC adresy

! Statická vazba MAC → IP
ip dhcp pool TISKARNA
  host 192.168.1.100 255.255.255.0        ! konkrétní IP
  client-identifier 01aa.bbcc.ddee.ff      ! MAC adresa (01 = Ethernet)
  ! nebo:
  hardware-address aabb.ccdd.eeff          ! alternativní zápis
  default-router 192.168.1.1

Linux (ISC DHCP Server – dhcpd)

# /etc/dhcp/dhcpd.conf

subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.20 192.168.1.200;     # rozsah přidělovaných adres
    option routers 192.168.1.1;            # výchozí brána
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    option domain-name "firma.local";
    default-lease-time 600;                # 10 minut
    max-lease-time 7200;                   # 2 hodiny
}

# Rezervace dle MAC
host tiskarna {
    hardware ethernet AA:BB:CC:DD:EE:FF;
    fixed-address 192.168.1.100;
}

DHCP Relay (ip helper-address)

DHCP Discover je broadcast → neprojde přes router. DHCP Relay přeposílá DHCP zprávy z jedné podsítě na DHCP server v jiné podsíti.

! Na rozhraní routeru směrem ke klientské síti
interface GigabitEthernet0/0
  ip helper-address 10.0.0.5              ! IP adresa DHCP serveru v jiné síti

DHCP Snooping (bezpečnost)

Ochrana proti podvrženým DHCP serverům (DHCP Spoofing).

ip dhcp snooping
ip dhcp snooping vlan 10,20

! Důvěryhodný port (kde je skutečný DHCP server)
interface GigabitEthernet0/1
  ip dhcp snooping trust

! Nedůvěryhodné porty (klienti) – defaultně untrusted
! Omezení rychlosti DHCP zpráv
interface range FastEthernet0/1-24
  ip dhcp snooping limit rate 10          ! max 10 DHCP paketů/s
