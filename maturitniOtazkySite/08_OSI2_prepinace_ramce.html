08 – OSI 2 – Linková vrstva – Přepínače, rámce, adresování, kolize, MAC, řízení toku

Linková vrstva (Data Link Layer)

Druhá vrstva OSI modelu. Zajišťuje spolehlivý přenos rámců mezi přímo spojenými uzly. Dělí se na dvě podvrstvy:

  • LLC (Logical Link Control) – IEEE 802.2 – multiplexování protokolů, řízení toku
  • MAC (Media Access Control) – IEEE 802.3 – přístup k médiu, adresování, rámcování

MAC adresa

  • 48 bitů (6 bajtů), zapsáno hexadecimálně: AA:BB:CC:DD:EE:FF
  • Prvních 24 bitů = OUI (Organizationally Unique Identifier) – identifikuje výrobce
  • Posledních 24 bitů = unikátní číslo zařízení
  • Unicast – nejnižší bit prvního bajtu = 0 (adresa jednoho zařízení)
  • Multicast – nejnižší bit prvního bajtu = 1
  • Broadcast – FF:FF:FF:FF:FF:FF (všechna zařízení v segmentu)
# Zobrazení MAC adresy
ip link show           # Linux
ipconfig /all          # Windows
ifconfig               # starší Linux/macOS

Přepínače (Switches)

Princip činnosti

  1. Učení (Learning) – switch si zapamatuje zdrojovou MAC adresu a port, ze kterého rámec přišel → MAC tabulka (CAM)
  2. Přeposílání (Forwarding) – má-li cílovou MAC v tabulce, pošle rámec jen na daný port
  3. Zaplavení (Flooding) – nemá-li cílovou MAC, pošle rámec na všechny porty kromě zdrojového
  4. Filtrování (Filtering) – pokud jsou zdrojový i cílový port stejné, rámec se zahodí
  5. Stárnutí (Aging) – nepoužívané záznamy v MAC tabulce se po čase mažou (výchozí 300 s)

Kolize

Kolizní doména

Část sítě, kde se mohou rámce srazit (kolize). Každý port switche = samostatná kolizní doména. Hub sdílí jednu kolizní doménu pro všechny porty.

Broadcast doména

Část sítě, kam dorazí broadcast rámec. Všechny porty switche = jedna broadcast doména. Router odděluje broadcast domény.

CSMA/CD (u hubů / half-duplex)

  1. Naslouchej – je médium volné?
  2. Ano → vysílej
  3. Detekuješ kolizi? → pošli JAM signál, čekej náhodný čas (exponenciální backoff)
  4. Zkus znovu (max. 16 pokusů)

Detekce a oprava chyb

FCS (Frame Check Sequence)

  • Každý Ethernet rámec obsahuje CRC-32 kontrolní součet
  • Odesílatel vypočítá CRC z obsahu rámce a přidá na konec
  • Příjemce přepočítá CRC a porovná → nesouhlasí = chyba → rámec zahozen
  • Linková vrstva neřeší opravu chyb – to dělá vyšší vrstva (TCP)

Řízení toku dat (Flow Control)

IEEE 802.3x – Pause Frames

Pokud je příjemce zahlcen, pošle PAUSE rámec odesílateli → ten pozastaví vysílání na specifikovanou dobu.

Priority Flow Control (PFC) – 802.1Qbb

Umožňuje pozastavení konkrétní priority provozu (pro lossless Ethernet – FCoE, iSCSI).

Přepínač – konfigurace (Cisco)

enable                              # privilegovaný režim
configure terminal                   # konfigurační režim

# Základní nastavení
hostname MujSwitch
enable secret MojeHeslo
service password-encryption

# Nastavení portu
interface FastEthernet0/1
  description PC-Uzivatel
  speed 100
  duplex full
  no shutdown

# Zabezpečení portů (Port Security)
interface FastEthernet0/1
  switchport mode access
  switchport port-security
  switchport port-security maximum 2
  switchport port-security mac-address sticky
  switchport port-security violation shutdown

show interfaces status
