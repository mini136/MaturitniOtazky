VLAN (Virtual LAN)

Co je VLAN

VLAN je logické rozdělení switche na více nezávislých broadcast domén. Fyzicky jeden switch, logicky několik oddělených sítí.

Proč používat VLAN

  • Segmentace – oddělení provozu (IT, účetnictví, management, hosté)
  • Bezpečnost – oddělení citlivých data bez fyzického oddělení
  • Omezení broadcastů – broadcast se šíří jen v rámci jedné VLAN
  • Flexibilita – přesunutí uživatele do jiné VLAN bez přepojování kabelů
  • Výkon – menší broadcast domény = menší zatížení sítě

Typy VLAN

TypPopis
Datová VLANBěžná VLAN pro uživatelský provoz
Native VLANVLAN pro netagované rámce na trunku (výchozí: VLAN 1)
Management VLANPro správu switche (SSH, SNMP) – z bezpečnostních důvodů ne VLAN 1
Voice VLANSpeciální VLAN pro VoIP telefony (QoS)
VLAN 1Výchozí VLAN – všechny porty jsou v ní defaultně; nelze smazat

Access a Trunk porty

Access port

  • Patří do jedné VLAN
  • Rámce nejsou tagované (koncové zařízení neví o VLAN)
  • Typicky připojení: PC, tiskárna, server

Trunk port

  • Přenáší provoz více VLAN na jednom spoji
  • Rámce jsou tagované identifikátorem VLAN (802.1q)
  • Typicky: propojení switch–switch, switch–router

Konfigurace (Cisco)

! Vytvoření VLAN
vlan 10
  name IT
vlan 20
  name Ucetnictvi
vlan 30
  name Hoste

! Access port
interface FastEthernet0/1
  switchport mode access
  switchport access vlan 10
  no shutdown

! Voice VLAN (IP telefon + PC za ním)
interface FastEthernet0/5
  switchport mode access
  switchport access vlan 10
  switchport voice vlan 50
  no shutdown

! Trunk port
interface GigabitEthernet0/1
  switchport mode trunk
  switchport trunk encapsulation dot1q    ! u starších switchů
  switchport trunk allowed vlan 10,20,30
  switchport trunk native vlan 99
  no shutdown

! Ověření
show vlan brief
show interfaces trunk
show interfaces FastEthernet0/1 switchport

IEEE 802.1Q

Princip tagování

802.1Q vkládá 4bajtový tag do Ethernet rámce mezi zdrojovou MAC adresu a Type pole.

Struktura 802.1Q tagu

PoleBityPopis
TPID (Tag Protocol ID)16Hodnota 0x8100 – identifikace 802.1Q tagu
PCP (Priority Code Point)3Priorita rámce (0–7) pro QoS (802.1p)
DEI (Drop Eligible Indicator)1Indikátor, že rámec může být zahozen
VID (VLAN ID)12Číslo VLAN (0–4095, použitelné 1–4094)

Tagovaný vs. netagovaný rámec

Normální Ethernet rámec:
[Dest MAC][Src MAC][Type][Data][FCS]

802.1Q tagovaný rámec:
[Dest MAC][Src MAC][0x8100][TCI (PCP+DEI+VID)][Type][Data][FCS]
                   └────── 802.1Q tag (4 B) ──────┘

Native VLAN

  • Rámce v native VLAN se na trunku posílají bez tagu
  • Native VLAN musí být stejná na obou koncích trunku
  • Výchozí: VLAN 1 – doporučuje se změnit (bezpečnost)
VLAN Hopping útok: Útočník zneužije výchozí native VLAN 1 k přeskočení do jiné VLAN. Obrana: změnit native VLAN, nepoužívat VLAN 1, nastavit switchport trunk native vlan 999.

DTP (Dynamic Trunking Protocol)

Co je DTP

Cisco proprietární protokol pro automatické vyjednávání trunk/access režimu portů.

Režimy DTP

RežimPopis
dynamic autoPasivně čeká na vyjednávání trunku (výchozí na většině switchů)
dynamic desirableAktivně se pokouší vyjednat trunk
trunkVždy trunk – i odešle DTP, ale i bez vyjednávání
accessVždy access – DTP se stále posílá!

Matice výsledků

autodesirabletrunkaccess
autoaccesstrunktrunkaccess
desirabletrunktrunktrunkaccess
trunktrunktrunktrunk—
accessaccessaccess—access

Vypnutí DTP (doporučeno z bezpečnostních důvodů)

interface FastEthernet0/1
  switchport mode access
  switchport nonegotiate        ! vypnutí DTP

interface GigabitEthernet0/1
  switchport mode trunk
  switchport nonegotiate        ! vypnutí DTP i na trunku
Bezpečnost: Vždy vypínejte DTP (switchport nonegotiate)! Útočník může pomocí DTP se switchem vyjednat trunk a získat přístup ke všem VLAN.

Inter-VLAN Routing

Komunikace mezi VLAN vyžaduje L3 zařízení (router).

Router-on-a-stick

! Router – subinterfaces
interface GigabitEthernet0/0.10
  encapsulation dot1Q 10
  ip address 192.168.10.1 255.255.255.0

interface GigabitEthernet0/0.20
  encapsulation dot1Q 20
  ip address 192.168.20.1 255.255.255.0

interface GigabitEthernet0/0
  no shutdown

L3 Switch (SVI – Switch Virtual Interface)

ip routing                    ! zapnutí směrování

interface vlan 10
  ip address 192.168.10.1 255.255.255.0
  no shutdown

interface vlan 20
  ip address 192.168.20.1 255.255.255.0
  no shutdown
