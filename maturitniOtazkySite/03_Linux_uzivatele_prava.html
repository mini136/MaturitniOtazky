03 – OS Linux – Správa uživatelů, skupin, zabezpečení přístupu, práva, SUDO

Uživatelé

Typy uživatelů

  • root (UID 0) – superuživatel, má plný přístup ke všemu
  • Systémoví uživatelé (UID 1–999) – pro služby (www-data, mysql, nobody)
  • Běžní uživatelé (UID 1000+) – lidské účty

Důležité soubory

SouborObsah
/etc/passwdSeznam uživatelů – username:x:UID:GID:info:home:shell
/etc/shadowHashovaná hesla, politika hesel (čitelný jen root)
/etc/groupSkupiny – groupname:x:GID:členové
/etc/gshadowHesla skupin
/etc/skelŠablona domovského adresáře nového uživatele
/etc/login.defsVýchozí nastavení pro nové uživatele

Formát /etc/passwd

username:x:1000:1000:Celé Jméno:/home/username:/bin/bash
# x = heslo je v /etc/shadow
# UID = User ID
# GID = primární Group ID

Správa uživatelů

# Vytvoření uživatele
useradd -m -s /bin/bash -G sudo,docker novyuser
# -m = vytvořit domovský adresář
# -s = výchozí shell
# -G = doplňkové skupiny

# Nastavení hesla
passwd novyuser

# Úprava uživatele
usermod -aG docker novyuser      # přidání do skupiny (-a = append!)
usermod -s /bin/zsh novyuser     # změna shellu
usermod -L novyuser              # zamčení účtu
usermod -U novyuser              # odemčení účtu

# Smazání uživatele
userdel novyuser                 # smazání (zachová /home)
userdel -r novyuser              # smazání včetně /home

# Informace o uživateli
id novyuser                      # UID, GID, skupiny
whoami                           # aktuální uživatel
who / w                          # kdo je přihlášen
last                             # historie přihlášení

Skupiny

# Vytvoření skupiny
groupadd vyvojari

# Přidání uživatele do skupiny
usermod -aG vyvojari novyuser    # POZOR: bez -a přepíše skupiny!
gpasswd -a novyuser vyvojari    # alternativa

# Odebrání ze skupiny
gpasswd -d novyuser vyvojari

# Smazání skupiny
groupdel vyvojari

# Výpis skupin uživatele
groups novyuser
id novyuser

Primární vs. doplňkové skupiny

  • Primární skupina – určena GID v /etc/passwd, nově vytvořené soubory patří této skupině
  • Doplňkové (suplementární) skupiny – další skupiny, do kterých uživatel patří

Práva (Permissions)

Základní práva

PrávoZnakČíselněPro souborPro adresář

Writew2Zápis/úpravaVytvoření/smazání souborů uvnitř
Executex1SpuštěníVstup do adresáře (cd)

Výpis práv (ls -l)

-rwxr-xr-- 1 user group 4096 Jan 01 12:00 soubor.sh
│││││││││
│└┬┘└┬┘└┬┘
│ │  │  └── ostatní (others): r-- = 4
│ │  └───── skupina (group):  r-x = 5
│ └──────── vlastník (user):  rwx = 7
└────────── typ souboru: - = soubor, d = adresář, l = link

Změna práv – chmod

# Číselně (oktálově)
chmod 755 soubor.sh      # rwxr-xr-x
chmod 644 soubor.txt     # rw-r--r--
chmod 700 privatni/      # rwx------

# Symbolicky
chmod u+x soubor.sh      # přidat execute pro vlastníka
chmod g-w soubor.txt     # odebrat write pro skupinu
chmod o=r soubor.txt     # nastavit ostatním jen read
chmod a+r soubor.txt     # přidat read pro všechny (all)
chmod -R 755 adresar/    # rekurzivně

Změna vlastníka – chown

chown user soubor              # změna vlastníka
chown user:group soubor        # vlastník + skupina
chown :group soubor            # jen skupina
chown -R user:group adresar/   # rekurzivně

Změna skupiny – chgrp

chgrp group soubor

Speciální bity

BitČíselněPopis
SUID4xxxSoubor se spustí s právy vlastníka (např. /usr/bin/passwd)
SGID2xxxSoubor: spuštění s právy skupiny. Adresář: nové soubory dědí skupinu
Sticky bit1xxxV adresáři může soubor smazat jen vlastník (např. /tmp)
chmod 4755 program       # SUID
chmod 2755 adresar/      # SGID
chmod 1777 /tmp          # Sticky bit
chmod u+s program        # SUID symbolicky
chmod g+s adresar/       # SGID symbolicky
chmod +t adresar/        # Sticky bit symbolicky

Výchozí práva – umask

umask                    # zobrazí aktuální masku (typicky 0022)
umask 0027               # nastavení masky

# Výchozí práva = maximální - umask
# Soubory: 666 - 022 = 644 (rw-r--r--)
# Adresáře: 777 - 022 = 755 (rwxr-xr-x)

ACL (Access Control Lists)

Rozšířená práva nad rámec standardního Unix modelu owner/group/others.

setfacl -m u:novyuser:rwx soubor    # přidání ACL pro uživatele
setfacl -m g:vyvojari:rx soubor     # přidání ACL pro skupinu
getfacl soubor                       # zobrazení ACL
setfacl -x u:novyuser soubor        # odebrání ACL
setfacl -b soubor                    # smazání všech ACL

SUDO

Co je SUDO

sudo (Substitute User DO) – umožňuje oprávněným uživatelům spouštět příkazy jako root (nebo jiný uživatel) bez znalosti root hesla.

Konfigurace – /etc/sudoers

# Editace VŽDY přes visudo (kontrola syntaxe!)
visudo

# Formát: kdo   kde=(jako kdo)   co
root    ALL=(ALL:ALL) ALL          # root může vše
%sudo   ALL=(ALL:ALL) ALL          # skupina sudo může vše
novyuser ALL=(ALL) NOPASSWD: ALL   # bez hesla (nebezpečné!)
novyuser ALL=(ALL) /usr/bin/apt    # jen příkaz apt

# Include soubory
# Práva se dají definovat v /etc/sudoers.d/
# Např. /etc/sudoers.d/novyuser

Použití sudo

sudo apt update              # spuštění jako root
sudo -u jinyuser příkaz      # spuštění jako jiný uživatel
sudo -i                      # interaktivní root shell
sudo -l                      # výpis oprávnění aktuálního uživatele
sudo su -                    # přepnutí na root účet
Bezpečnost: SUDO loguje všechny příkazy. Nikdy nedávejte NOPASSWD: ALL běžným uživatelům. Vždy specifikujte minimální nutná oprávnění.

Zabezpečení přístupu ke službám

  • PAM (Pluggable Authentication Modules) – modulární autentizační framework
  • Konfigurace v /etc/pam.d/
  • TCP Wrappers – /etc/hosts.allow, /etc/hosts.deny
  • Firewall (iptables/nftables/ufw) – omezení síťového přístupu
  • SELinux / AppArmor – mandatory access control (MAC)
