Symetrické šifrování

Princip

Stejný klíč pro šifrování i dešifrování. Rychlé, ale problém s distribucí klíče.

Plaintext → [Šifrování, klíč K] → Ciphertext → [Dešifrování, klíč K] → Plaintext

Asymetrické šifrování

Princip

Dva klíče: veřejný (šifrování) a privátní (dešifrování). Řeší problém distribuce klíčů.

Šifrování:    Plaintext → [Šifrování, veřejný klíč příjemce] → Ciphertext
Dešifrování:  Ciphertext → [Dešifrování, privátní klíč příjemce] → Plaintext

Digitální podpis:
  Podpis:     Hash(data) → [Šifrování, privátní klíč odesílatele] → Podpis
  Ověření:    Podpis → [Dešifrování, veřejný klíč odesílatele] → Hash → porovnání

RSA (Rivest–Shamir–Adleman)

  • Nejznámější asymetrický algoritmus
  • Bezpečnost založena na obtížnosti faktorizace velkých prvočísel
  • Typicky 2048 nebo 4096 bit klíč
  • Pomalejší než symetrické → používá se pro výměnu klíčů a digitální podpisy

Princip RSA (zjednodušeně)

1. Zvol 2 velká prvočísla: p, q
2. Vypočítej n = p × q (modulus)
3. Vypočítej φ(n) = (p-1)(q-1)
4. Zvol veřejný exponent e (typicky 65537)
5. Vypočítej privátní exponent d: e × d ≡ 1 (mod φ(n))

Veřejný klíč:  (n, e)
Privátní klíč: (n, d)

Šifrování:   C = M^e mod n
Dešifrování: M = C^d mod n

SSL/TLS

Přehled

  • SSL (Secure Sockets Layer) – zastaralý, vytvořen Netscape (SSL 2.0, 3.0)
  • TLS (Transport Layer Security) – nástupce SSL, aktuální verze TLS 1.3
  • Zajišťuje: důvěrnost (šifrování), integritu (MAC/HMAC), autenticitu (certifikáty)

TLS Handshake (TLS 1.2)

Klient                                    Server
  |                                          |
  |-- ClientHello --------------------------->|  Verze, cipher suites, random
  |                                          |
  |<--- ServerHello + Certificate -----------|  Vybraný cipher, certifikát (veřejný klíč)
  |<--- ServerKeyExchange (pokud DH) --------|
  |<--- ServerHelloDone ---------------------|
  |                                          |
  |-- ClientKeyExchange --------------------->|  Pre-master secret (šifrovaný veřejným klíčem)
  |-- ChangeCipherSpec ---------------------->|  "Přepínám na šifrování"
  |-- Finished (šifrovaně) ----------------->|
  |                                          |
  |<--- ChangeCipherSpec --------------------|
  |<--- Finished (šifrovaně) ----------------|
  |                                          |
  |===== ŠIFROVANÁ KOMUNIKACE ==============|

2 RTT (round trips) pro TLS 1.2

TLS 1.3 Handshake (zrychlený)

Klient                                    Server
  |                                          |
  |-- ClientHello + KeyShare ---------------->|  1 RTT!
  |                                          |
  |<--- ServerHello + KeyShare --------------|
  |<--- EncryptedExtensions + Certificate ---|
  |<--- Finished ----------------------------|
  |                                          |
  |-- Finished ------------------------------>|
  |                                          |
  |===== ŠIFROVANÁ KOMUNIKACE ==============|

Pouze 1 RTT! (+ 0-RTT resumption – volitelný)

Vylepšení TLS 1.3

  • Pouze bezpečné cipher suites (AEAD: AES-GCM, ChaCha20-Poly1305)
  • Odstraněny: RSA key exchange (bez forward secrecy), CBC režim, SHA-1, RC4, 3DES
  • Povinný ECDHE → Perfect Forward Secrecy
  • 1 RTT handshake (oproti 2 RTT v TLS 1.2)

X.509 certifikát obsahuje

  • Doménové jméno (Subject / SAN)
  • Veřejný klíč serveru

Let's Encrypt

Bezplatná automatická CA – ACME protokol, DV (Domain Validation) certifikáty.

Cipher Suite (příklad TLS 1.2)

TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384

TLS       – protokol
ECDHE     – key exchange (Ephemeral Elliptic Curve Diffie-Hellman)
RSA       – autentizace (podpis certifikátu)
AES_256   – symetrické šifrování (256-bit)
GCM       – režim šifrování (Galois/Counter Mode → AEAD)
SHA384    – hash pro HMAC / PRF
Pro maturitu: Symetrické = AES (rychlé, šifrování dat). Asymetrické = RSA/ECDH (výměna klíčů, podpisy). TLS = kombinace obou. TLS 1.3 = 1 RTT, povinný forward secrecy. Certifikát = veřejný klíč serveru podepsaný CA.
