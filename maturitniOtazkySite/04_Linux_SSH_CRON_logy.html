SSH (Secure Shell)

Základní princip

SSH je protokol pro šifrované vzdálené připojení ke stroji. Běží na portu 22/TCP. Nahrazuje nezabezpečené protokoly (telnet, rlogin).

Základní použití

# Připojení
ssh user@server                     # základní připojení
ssh -p 2222 user@server             # jiný port
ssh user@server 'ls /var/log'       # spuštění příkazu na vzdáleném stroji

# Kopírování souborů
scp soubor.txt user@server:/tmp/    # kopie na server
scp user@server:/var/log/syslog .   # kopie ze serveru
scp -r adresar/ user@server:/tmp/   # rekurzivní kopie

# SFTP
sftp user@server                     # interaktivní přenos souborů

Autentizace klíčem

# Generování páru klíčů
ssh-keygen -t ed25519 -C "email@domena.cz"
# Privátní klíč: ~/.ssh/id_ed25519
# Veřejný klíč: ~/.ssh/id_ed25519.pub

# Kopírování veřejného klíče na server
ssh-copy-id user@server
# Ekvivalent: cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys

# Nyní se přihlásíme bez hesla

Konfigurace SSH serveru

# /etc/ssh/sshd_config
Port 22                          # port
PermitRootLogin no               # zakázání přímého root přístupu
PasswordAuthentication no        # pouze klíče (bezpečnější)
PubkeyAuthentication yes
AllowUsers user1 user2           # povolení jen konkrétním uživatelům
MaxAuthTries 3                   # max pokusů o přihlášení

# Restart po změně
systemctl restart sshd

SSH klientská konfigurace (~/.ssh/config)

Host mujserver
    HostName 192.168.1.100
    User admin
    Port 2222
    IdentityFile ~/.ssh/id_ed25519
# Po konfiguraci stačí: ssh mujserver

SSH tunelování

# Local port forwarding – přesměrování lokálního portu na vzdálený
ssh -L 8080:localhost:80 user@server    # localhost:8080 → server:80

# Remote port forwarding
ssh -R 8080:localhost:80 user@server    # server:8080 → localhost:80

# SOCKS proxy
ssh -D 1080 user@server                 # dynamický tunel

CRON – Plánování úloh

Crontab

# Editace cronu aktuálního uživatele
crontab -e

# Výpis cronu
crontab -l

# Formát:
# MIN HOUR DOM MON DOW COMMAND
# 0-59 0-23 1-31 1-12 0-7(0,7=Ne)

# Příklady:
0 2 * * * /opt/backup.sh           # každý den ve 2:00
*/15 * * * * /opt/check.sh          # každých 15 minut
0 0 * * 0 /opt/weekly.sh            # každou neděli o půlnoci
30 8 1 * * /opt/monthly.sh          # 1. den v měsíci v 8:30
0 6 * * 1-5 /opt/workday.sh         # Po-Pá v 6:00

Speciální řetězce

@reboot     /opt/startup.sh          # při startu systému
@hourly     /opt/hourly.sh           # každou hodinu
@daily      /opt/daily.sh            # denně
@weekly     /opt/weekly.sh           # týdně
@monthly    /opt/monthly.sh          # měsíčně

Systémový cron

# /etc/crontab – systémový crontab (navíc pole pro uživatele)
# /etc/cron.d/ – další cron soubory
# /etc/cron.daily/ – skripty spouštěné denně (anacron)
# /etc/cron.hourly/
# /etc/cron.weekly/
# /etc/cron.monthly/

Systemd timery (alternativa k cronu)

# /etc/systemd/system/muj-backup.timer
[Unit]
Description=Backup timer

[Timer]
OnCalendar=daily
Persistent=true

[Install]
WantedBy=timers.target

# Správa
systemctl enable --now muj-backup.timer
systemctl list-timers

Procesy a služby

Správa procesů

ps aux                      # všechny procesy
ps -ef --forest             # stromová struktura
top / htop                  # interaktivní monitor
pgrep -a nginx              # hledání procesu podle jména
pidof sshd                  # PID procesu

# Signály
kill PID                    # SIGTERM (15) – slušné ukončení
kill -9 PID                 # SIGKILL – vynucené ukončení
kill -HUP PID               # SIGHUP – reload konfigurace
killall nginx               # zabití všech dle jména
pkill -f "python script"    # zabití dle vzoru

# Priorita
nice -n 10 command          # spuštění s nižší prioritou (-20 až 19)
renice -n 5 -p PID          # změna priority běžícího procesu

Služby (systemd)

systemctl start/stop/restart/reload služba
systemctl enable/disable služba          # autostart
systemctl status služba                  # stav
systemctl is-active služba               # běží?
systemctl is-enabled služba              # povolen autostart?
systemctl list-units --type=service      # seznam služeb
systemctl list-unit-files --type=service # všechny unit soubory
systemctl daemon-reload                  # po změně unit souborů
journalctl -u služba -f                  # logy služby (živě)

Systémové logy

Journald (systemd)

journalctl                          # všechny logy
journalctl -u sshd                  # logy konkrétní služby
journalctl -f                       # živé sledování (jako tail -f)
journalctl --since "2024-01-01"     # od data
journalctl --since "1 hour ago"     # za poslední hodinu
journalctl -p err                   # jen chyby (emerg,alert,crit,err,warning,notice,info,debug)
journalctl -b                       # logy od posledního bootu
journalctl -b -1                    # logy z předchozího bootu
journalctl --disk-usage             # využití místa

Logrotate

Automatická rotace, komprese a mazání starých logů. Konfigurace: /etc/logrotate.conf a /etc/logrotate.d/

# /etc/logrotate.d/myapp
/var/log/myapp/*.log {
    daily
    rotate 7
    compress
    missingok
    notifempty
    postrotate
        systemctl reload myapp
    endscript
}

Virtualizace

Typy virtualizace

TypPopisPříklady
Plná virtualizaceKompletní emulace HW, nepotřebuje úpravu guest OSKVM, VirtualBox, VMware
ParavirtualizaceGuest OS je upraven pro spolupráci s hypervisoremXen (PV mode)
KontejnerizaceSdílí jádro hostitelského systému, izolované prostředíDocker, LXC, Podman

Hypervisory

  • Typ 1 (bare-metal) – běží přímo na HW: VMware ESXi, Xen, KVM
  • Typ 2 (hosted) – běží nad OS: VirtualBox, VMware Workstation

KVM (Kernel-based Virtual Machine)

# Kontrola podpory virtualizace
egrep -c '(vmx|svm)' /proc/cpuinfo   # > 0 = podpora

# Instalace
apt install qemu-kvm libvirt-daemon-system virt-manager

# Správa
virsh list --all              # seznam VM
virsh start vm-name           # spuštění
virsh shutdown vm-name        # vypnutí
virsh destroy vm-name         # vynucené vypnutí
virsh snapshot-create vm-name # snapshot

Docker (kontejnerizace)

# Základní příkazy
docker run -d --name web -p 80:80 nginx    # spuštění kontejneru
docker ps                                   # běžící kontejnery
docker stop/start/restart web               # správa
docker logs web                             # logy
docker exec -it web bash                    # vstup do kontejneru
docker images                               # lokální obrazy
docker build -t myapp .                     # sestavení obrazu
Kontejnery vs. VM: Kontejnery sdílejí jádro → menší overhead, rychlejší start, ale menší izolace. VM mají vlastní jádro → vyšší izolace, ale větší spotřeba zdrojů.
