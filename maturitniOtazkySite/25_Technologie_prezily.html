25 – Technologie, které přežily změny současného rozvoje ICT

Ethernet (1973 → dodnes)

VlastnostPopis
Vznik1973 – Robert Metcalfe, Xerox PARC
StandardIEEE 802.3
Původní rychlost2,94 Mbps (koax)
Dnes400 Gbps (802.3bs), vývoj 800G/1,6T
Proč přežilJednoduchost, zpětná kompatibilita, škálovatelnost, nízká cena

Evoluce

  • 10BASE5 (thick coax) → 10BASE-T (UTP, hvězdicová topologie) → 100BASE-TX (Fast Ethernet)
  • 1000BASE-T (Gigabit) → 10GBASE-T → 25G/40G/100G (data centra)
  • Přechod z koaxu na UTP, pak na optiku v páteřních sítích
  • CSMA/CD → full duplex (bez kolizí) u přepínaných sítí

TCP/IP (1974 → dodnes)

VlastnostPopis
Vznik1974 – Vint Cerf, Bob Kahn (ARPANET)
RFCTCP: RFC 793 (1981), IP: RFC 791 (1981)
DnesZáklad celého internetu a většiny sítí
Proč přežilOtevřené standardy, vrstvová architektura, nezávislost na médiu

Klíčové principy

  • End-to-end princip – inteligence na koncových bodech, síť je „hloupá" (best effort)
  • Vrstvový model – každá vrstva řeší svůj problém
  • Packet switching – data rozdělena na pakety, nezávislé směrování
  • IPv4 → IPv6 přechod zajišťuje budoucnost

DNS (1983 → dodnes)

VlastnostPopis
Vznik1983 – Paul Mockapetris (RFC 882/883, poté RFC 1034/1035)
NahradilHOSTS.TXT – centrální soubor na SRI-NIC (neškálovatelný)
DnesHierarchický distribuovaný systém, 13 root serverů (anycast)
Proč přežilDistribuovaná architektura, caching, hierarchie, rozšiřitelnost

Evoluce

  • Původní DNS → DNSSEC (podpisy, ochrana integrity)
  • DoH (DNS over HTTPS) a DoT (DNS over TLS) – šifrování dotazů
  • Nové TLD (.app, .dev, .io…)
  • Cloudflare (1.1.1.1), Google (8.8.8.8) – veřejné resolvery

SMTP (1982 → dodnes)

VlastnostPopis
Vznik1982 – RFC 821 (Jon Postel)
DnesRFC 5321 (2008), stále základ e-mailové komunikace
Proč přežilJednoduchost, rozšiřitelnost (ESMTP), store-and-forward model

Evoluce

  • Původní plain text → STARTTLS (šifrování)
  • SPF, DKIM, DMARC – ochrana proti spamu a spoofingu
  • MIME – přílohy, HTML e-maily, kódování
  • I přes alternativy (messaging, Slack…) e-mail zůstává klíčový

HTTP/HTML/WWW (1991 → dodnes)

VlastnostPopis
Vznik1989–1991 – Tim Berners-Lee (CERN)
Verze HTTP0.9 → 1.0 → 1.1 → 2 (2015) → 3 (2022, QUIC)
Proč přežilOtevřené standardy, jednoduchost, hyperlinkový model

Evoluce

  • HTTP/1.0 (jedno spojení na požadavek) → HTTP/1.1 (keep-alive, pipelining)
  • HTTP/2 (multiplexing, binární, header compression, server push)
  • HTTP/3 (UDP + QUIC – 0-RTT, bez head-of-line blocking)
  • HTML 1.0 → HTML5 (multimedia, canvas, WebSocket, offline apps)
  • HTTPS (TLS) – dnes default (Let's Encrypt)

BGP (1989 → dodnes)

VlastnostPopis
Vznik1989 – RFC 1105, aktuální BGP-4 RFC 4271
FunkceSměrování mezi autonomními systémy (inter-AS / EGP)
Dnes„Lepidlo internetu" – propojuje ~70 000+ AS
Proč přežilPolitika směrování, škálovatelnost, flexibilita

SSH (1995 → dodnes)

VlastnostPopis
Vznik1995 – Tatu Ylönen (nahradil Telnet, rlogin, rsh)
StandardRFC 4251–4254 (SSH-2)
DnesVzdálená správa serverů, přenos souborů, tunely, Git
Proč přežilBezpečnost, všestrannost, OpenSSH implementace

DHCP (1993 → dodnes)

VlastnostPopis
Vznik1993 – RFC 1531 (nahradil BOOTP)
DnesDHCPv4 (RFC 2131) + DHCPv6 (RFC 8415)
Proč přežilAutomatická konfigurace, bez alternativy pro IPv4

Wi-Fi / IEEE 802.11 (1997 → dodnes)

VlastnostPopis
Vznik1997 – IEEE 802.11 (2 Mbps, 2.4 GHz)
DnesWi-Fi 7 (802.11be, 46 Gbps, 2.4/5/6 GHz)
Proč přežilMobilita, pohodlí, zpětná kompatibilita, neustálé inovace

Proč některé technologie přežijí

  1. Otevřené standardy – TCP/IP, HTTP, SMTP jsou otevřené a svobodné
  2. Zpětná kompatibilita – Ethernet, Wi-Fi – novější verze podporují starší
  3. Škálovatelnost – DNS hierarchie, TCP/IP vrstvový model
  4. Adaptabilita – HTTP (text → binary → QUIC), SMTP (plain → TLS + DKIM)
  5. Síťový efekt – čím více uživatelů, tím hodnotnější (internet, e-mail)
  6. Jednoduchost – End-to-end princip, minimální jádro, rozšiřitelnost
Pro maturitu: Přežily: Ethernet, TCP/IP, DNS, SMTP, HTTP, BGP, SSH, DHCP, Wi-Fi. Nepřežily: Token Ring, ATM, IPX/SPX, Telnet, ISDN. Klíč k přežití = otevřené standardy, zpětná kompatibilita, škálovatelnost, adaptabilita.
