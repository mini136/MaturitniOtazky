ARP (Address Resolution Protocol)

Účel

ARP překládá IP adresy (L3) na MAC adresy (L2) v rámci jedné lokální sítě. Bez ARP by stanice nevěděla, na jakou MAC adresu poslat Ethernet rámec.

Princip ARP

  1. Stanice A chce komunikovat s IP 192.168.1.10, ale nezná jeho MAC adresu
  2. A pošle ARP Request (broadcast FF:FF:FF:FF:FF:FF): „Kdo má IP 192.168.1.10? Řekněte to MAC adrese AA:BB:CC:DD:EE:01"
  3. Všechny stanice v segmentu přijmou broadcast, ale odpovídá jen ta s danou IP
  4. Stanice B s IP 192.168.1.10 odpoví ARP Reply (unicast): „192.168.1.10 jsem já, moje MAC je AA:BB:CC:DD:EE:02"
  5. Stanice A uloží záznam do ARP cache

ARP tabulka (ARP Cache)

# Zobrazení ARP tabulky
arp -a                   # Windows i Linux
ip neigh show            # Linux (moderní)

# Příklad výstupu:
# 192.168.1.1    00:1a:2b:3c:4d:5e    dynamic
# 192.168.1.10   aa:bb:cc:dd:ee:02    dynamic

# Smazání záznamu
arp -d 192.168.1.10      # smazání konkrétního záznamu
ip neigh flush all        # vymazání všech (Linux)

Typy záznamů

  • Dynamic – naučený přes ARP, vypršení obvykle po 1–5 minutách
  • Static – ručně zadaný, nevyprší

ARP útoky

  • ARP Spoofing/Poisoning – útočník posílá falešné ARP Reply → otrávení ARP cache obětí → Man-in-the-Middle
  • Obrana: Dynamic ARP Inspection (DAI) na switchi, statické ARP záznamy, VLAN segmentace

Gratuitous ARP

Stanice posílá ARP Request pro svou vlastní IP adresu. Účel: oznámení své MAC adresy sítě, detekce duplicitní IP adresy. Používá se při startu zařízení nebo po změně MAC (např. HSRP failover).

Proxy ARP

Router odpovídá na ARP Request za stanice v jiné síti, pokud zná cestu. Umožňuje komunikaci bez výchozí brány (nedoporučuje se).

Vztah mezi rámcem a paketem

Rámec obsahuje paket

Paket (L3) je zapouzdřen (encapsulován) uvnitř rámce (L2) jako jeho datová část (payload).

Zapouzdření (Encapsulation)

Proces zapouzdření

  1. Aplikační vrstva (L7/6/5) – data aplikace
  2. Transportní vrstva (L4) – přidá TCP/UDP hlavičku → Segment
  3. Síťová vrstva (L3) – přidá IP hlavičku → Paket
  4. Linková vrstva (L2) – přidá MAC hlavičku + FCS → Rámec
  5. Fyzická vrstva (L1) – převod na bity → elektrické/optické signály

Proces de-zapouzdření

Na přijímající straně probíhá opačný proces – každá vrstva odstraní svou hlavičku a předá data vyšší vrstvě.

Příklad: PC A (192.168.1.2) → Router → PC B (10.0.0.5)

HopZdrojová MACCílová MACZdrojová IPCílová IP
PC A → RouterMAC_AMAC_Router_in192.168.1.210.0.0.5
Router → PC BMAC_Router_outMAC_B192.168.1.210.0.0.5
Klíčový koncept: IP adresy se po cestě nemění (end-to-end), ale MAC adresy se mění na každém hopu (hop-by-hop). Router odstraní starý L2 rámec a vytvoří nový s novými MAC adresami.
